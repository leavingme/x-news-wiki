# WebMCP：Google 正在把网页变成 AI 可调用的 API

## 摘要

**[网站「AI 适配」将成生死线，流量迁移拐点已现]**

• Chrome 146 实验性推出 WebMCP，网站主动以结构化工具形式暴露能力，AI agent 直接调用而非截图猜按钮
• 联合微软推进 W3C 标准草案，支持 HTML 表单属性标注或 JS 注册复杂工具接口
• 类比移动互联网初期：不做移动适配的网站流量腰斩，AI agent 流量迁移正在重演

**gakki 锐评**：这不是技术优化，是权力转移——从「AI 适应网页」到「网页适应 AI」，搜索引擎逻辑正在被重写。

## 原文

AI agent 操作网页一直有个根本性的问题：它得像人一样"看"页面，截图识别按钮，猜输入框该填什么格式，猜提交逻辑是什么。慢，还费 token...

Google 在 Chrome 146 里推了 WebMCP，思路完全反过来。让网站主动把自己的能力以结构化工具的形式暴露出来，AI agent 直接调用，拿到结构化结果。

举个例子：以前 AI 要在机票网站搜航班，得找搜索框、猜日期格式、点按钮、等页面刷新、再解析结果。

现在网站直接注册一个 searchFlights(出发地, 目的地, 日期) 工具，AI 一次调用拿到 JSON，中间环节全部跳过。

这是 Google 和微软联合推的 W3C 标准草案，目前在实验阶段。两种接入方式：给现有 HTML 表单加属性（轻量），或者用 JS 注册更复杂的工具接口。

以后的网站不只是给人看的，还得给 AI 用。当年移动互联网起来的时候，不做移动适配的网站流量直接腰斩。AI agent 的流量迁移正在发生同样的事。
