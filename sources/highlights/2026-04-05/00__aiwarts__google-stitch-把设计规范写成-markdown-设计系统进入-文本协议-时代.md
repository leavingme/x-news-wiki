# Google Stitch 把设计规范写成 Markdown，设计系统进入「文本协议」时代

## 摘要

**[一句话核心判断]**

Google Stitch 把 55 家公司的设计语言（颜色/字体/组件/布局）写成 DESIGN.md 规范，放在项目根目录供 Claude Code 读取执行——设计规范从 Figma 截图时代正式进入「文本驱动」时代。

- **信息增量**：Figma 的截图/口头描述 → Claude Code 可解析的 Markdown 约束文件，前端开发一致性保障从「人治」变为「机治」
- **影响范围**：设计师→开发者的协作协议被重新定义，Design Token 概念获得 AI-Native 落地载体
- **边界**：依赖模型严格遵循文件约束，多风格混合场景仍有漂移风险

gakki：Stripe/Vercel/Notion 们集体「投降」把设计语言交出，本质是 AI 时代的话语权转移——谁掌握 Prompt，谁就掌握定义权。

## 原文

没想到Google Stitch这次放出来给Agent用的设计Markdown DESIGN.md 还有后续，

把Stripe，Vercel，Notion，Apple 55家公司的设计语言也做成md了，以前设计规范大多在 Figma，截图或者口头描述。现在它直接变成一个放在项目根目录的 Markdown，颜色、字体、组件、布局这些约束都能写进去。Claude Code先读这份文件，再开始写页面，前端一致性会稳很多。
