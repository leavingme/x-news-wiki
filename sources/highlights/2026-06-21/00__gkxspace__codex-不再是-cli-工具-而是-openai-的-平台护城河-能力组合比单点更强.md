# Codex 不再是 CLI 工具，而是 OpenAI 的"平台护城河"——能力组合比单点更强

## 摘要

**Codex 已经从代码生成工具演化为"能力组合平台"，单点能力都有竞品但组合后无对手。**

基于 tldraw 构建本地无限画布，通过 MCP 让 Codex 原生读写画布元素、按比例生图、自动插入。

作者梳理 Codex 当前能力栈：编程、Image 2 最强生图、内置浏览器、Computer Use、Record & Replay（录操作→Skill）、Sites 一句话部署带数据库网站、插件生态上百个应用仍在疯涨。

关键判断：当编程/生图/浏览器/录屏/部署被整合到同一平台，竞品护城河消失，用户离开成本陡增。

gakki 锐评：这是 OpenAI 用 Codex 做"超级 App"的早期信号——Agent 战争终局不是单点模型更强，而是"用户离不开的生态"。但插件生态的护城河很薄，Anthropic 已经在 Skills 路线上抄近道。

## 原文

执行力真猛，一天就开源了。

认真想了一下 Codex 现在的状态，越想越觉得这东西是一盘大棋：

1、编程能力，有。Image 2 最强生图，有。
2、内置浏览器能操作网页，有。Computer Use 能操作整个电脑，有。
3、Record & Replay 录一遍操作就变成可复用 Skill，有。
4、Sites 一句话部署带数据库的网站，也有。

插件生态上百个应用，目前还在疯涨。

这些能力单独拿出来都有竞品，但 Codex 如果把它们组合在一个平台里的，还有多少产品有护城河，那时候就只有"用户离不开的生态了吧"？
