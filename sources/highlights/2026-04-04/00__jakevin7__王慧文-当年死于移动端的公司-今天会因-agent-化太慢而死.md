# 王慧文：当年死于移动端的公司，今天会因 Agent 化太慢而死

## 摘要

**王慧文与卡比卡比饭局对话核心：应用层的 Agent 化转型，已经是和当年移动化转型同级的人口。**

王慧文认为：今天绝大多数应用面临的不是「要不要做 AI」，而是「能不能赶上 Agent 化这班车」。这和 2010 年代网页到移动端的转型逻辑完全一致——慢一拍的公司直接被淘汰。

对话还透露 OpenCLI 的定位：做 AI Agent 的 read & action 基建，拥抱 AI native。



经历过移动端转型的人看这个问题有天然的「穿越感」。王慧文不是空谈，他是真的踩过坑的人。这个判断比任何研究报告都有分量。问题是今天国内有多少公司有王慧文这种警觉度？

## 原文

今天和王慧文吃饭，聊到个很有意思的话题。
当年很多公司因为网页端到移动端转型太慢然后死了。
今天的公司，会不会因为agentic化太慢然后导致死亡呢？
关于目前时代的应用们未来的方向怎么走，这是个很有趣的话题。

另外被问到OpenCLI的发展目标的，其实我自己的想法很简单，OpenCLI 设计彻底拥抱 AI native & 想成为 AI agent 的 read & action 的基建。

另外就是我太能叭叭了，时间少了交流还不够，哈哈。要是还能多聊到点上面的这种话题应该会很有趣。经历过历史转型的人的看法和体会会深刻和不一样一些。
