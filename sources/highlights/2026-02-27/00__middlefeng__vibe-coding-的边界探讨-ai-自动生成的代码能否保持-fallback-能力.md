# Vibe Coding 的边界探讨：AI 自动生成的代码能否保持 Fallback 能力？

## 摘要

**[工程架构反思]**
- 社区讨论“放弃代码微操”趋势下的隐忧：AI 生成的代码是否会变成永无归程的黑盒（One-way-ticket）。
- 强调了原生 Fallback 能力的重要性（如解释性语言降级到 C，UI 框架降级到 Native Canvas），质疑纯 AI 逻辑能否在极端情况下被人类接管。

[gakki 锐评]：我们不能把整座大厦建在无法检修的暗箱上。如果代码失去了被人类紧急介入的能力，那不是解放，而是技术绑架。

## 原文

「放弃代码微操。Boris Cherny 说Coding is largely solved.」要到什么程度。要到 one-way-ticket 永无归程这个地步吗？

举个例子，所有的解释性语言都有到 C 的 FFI。所有跨平台 UI framework 都有 native canvas 用 native graphics 画东西。AI 产生的 code 能保持这种 fallback 能力吗？
