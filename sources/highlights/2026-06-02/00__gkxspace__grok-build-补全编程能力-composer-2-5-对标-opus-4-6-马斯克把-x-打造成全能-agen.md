# Grok Build 补全编程能力：Composer 2.5 对标 Opus 4.6，马斯克把 X 打造成全能 Agent 入口

## 摘要

**一句话核心判断**
Grok Build 通过接入 X 搜索 + Composer 2.5 编程 + Imagine 多模态生成，正在从聊天机器人进化为企业级 Agent 平台。

**信息增量**
- Composer 2.5 编程能力被评测对标 Opus 4.6 级别，Cursor 老用户确认可用性
- X 搜索能力已接入，实现社交信息 + 全网搜索的双重增强
- Imagine 生图/视频尺度放开，暗示 Grok 正在抢占 Midjourney 和 Sora 的市场
- 马斯克通过多能力集成正在把 X 变成一个超级 Agent 入口，而非单纯的社交平台

**gakki 锐评**
Grok 的快速迭代证明了一件事：模型能力差距可以通过工程集成快速抹平，而真正的护城河是数据闭环和分发渠道——X 恰好两者都有。

## 原文

Grok Build 算是被马斯克盘活了，X 蓝 V 千万别浪费，这段时间先是接入了 X 搜索能力，现在又把 Composer 2.5 接入了。

Cursor 的 Composer 2.5 编程跟 Opus 4.6 同档，我在 Cursor 用了挺久了，确实很能打。

Grok Build 现在既内置了 Imagine 生图生视频（尺度还是很大），推特搜索也很好用，现在编程这块也补上来了，老马牛逼😋
