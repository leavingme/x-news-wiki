# 宝玉反驳「把用户当弱智」：豆包/Telegram/Codex 都在用 Markdown，照样活得好好的

## 摘要

**[Markdown 在 C 端并未被判死刑]**
· 反对「为普通用户简化就是为他们好」的家长式产品哲学
· 三个反例：豆包、Telegram、Codex 都支持 Markdown，用户体量并未因此塌缩
· 潜台词：把 Markdown 当工程师专属是技术中心主义，普通用户的容错与学习能力被严重低估
· 关键命题：UX 护城河不在隐藏复杂度，而在尊重用户的学习曲线

宝玉把 chat-ui-durability 的讨论拉回到消费者产品的真实场景：当 Telegram 能让十亿人用上 Markdown，豆包的「完蛋论」就站不住。边界在于——Markdown 的留存成本和首次接触转化率之间仍是未知数，反例只证明「不会死」，没证明「因此更好」。

## 原文

不一定非把普通用户想的跟弱智一样，支持了 Markdown 普通用户就不会用了。
