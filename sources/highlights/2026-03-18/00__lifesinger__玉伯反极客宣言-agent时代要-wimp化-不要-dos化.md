# 玉伯反极客宣言：Agent时代要「WIMP化」不要「DOS化」

## 摘要

**[files-based不是最优解，Agent喜欢现代人喜欢的]**
• 极客工具（Markdown/GitHub/files-based）用户基数远小于普通用户，Agent设计应面向现代人而非DOS极客
• DOS当年无所不能，但在WIMP（窗口/图标/菜单/指针）面前沦为非遗用法
• 随着大模型提升，AI会越来越懂现代人，极客范式只是过渡态

**gakki锐评：** 这是对产品路径的清醒认知——做给Agent用的工具，本质是做人用的工具，「极客审美」往往是产品大众化的绊脚石。

## 原文

非常不认可把极客的用法推广到大众

至今不知道 md 的用户远远远远大于知道 md 怎么用的用户

至于 GitHub，则后续人类都不需要知道了，Agent 知道就行

files-based 也并非最优解，就如 DOS 当年什么都能干，但在 WIMP 面前，DOS 就变成了少量极客的非遗用法

Agent 更像人，人喜欢的，终究也会变成 Agent 更喜欢

一定要把 AI 当成现代人去看，而不是当成 DOS 时代的人去看

随着大模型的提升，AI 会越来越懂现代人，而不是 DOS 级的极客用户
