# Google 官方认领「5大 Agent Skill 设计模式」：Claude Code 的 SKILL.md 正在成为全球标准

## 摘要

**[基础设施拐点已至]**
• Google 一次性发布 5 个 Skill 设计模式（Tool Wrapper / Generator / Reviewer / Inversion / Pipeline），标志 SKILL.md 从社区实践升级为行业规范
• 30+ Agent 工具已统一格式，核心战场从「格式之争」转向「内容质量之争」
• 组合式架构成为共识：Pipeline 嵌套 Reviewer、Generator 前置 Inversion，告别单一 prompt 堆砌

这不仅是技术标准化，更是 Agent 工程范式的权力转移——Anthropic 定义了 CLI 形态，Google 正在接管 Skill 标准。

## 原文

Google今天放大招了！今天一次性发布了5个Agent Skill设计模式。

现在30多个Agent工具（Claude Code、Gemini CLI、Cursor）已经统一了SKILL.md格式，格式不再是问题——真正的挑战是里面写什么。

5个模式：

Tool Wrapper：给Agent装上某个库的专家知识，按需加载，不占context

Generator：模板+样式指南，每次输出结构一致

Reviewer：把检查清单外置，换一份清单就变成另一个审核专家

Inversion：先采访你，收集完所有需求才开始执行

Pipeline：强制分步骤，每步有检查点，不允许跳过
这5个模式可以组合使用——Pipeline里嵌Reviewer，Generator前加Inversion。

与其把复杂逻辑塞进一个system prompt，不如用对结构。
