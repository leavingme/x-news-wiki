# Codex GPT-5.5 在 Mac 开发场景输给 Opus 4.8：模型垂直场景能力差距正在拉大

## 摘要

**[一句话核心判断]**

宝玉实测发现 Codex 内置的 GPT-5.5 在 Mac 应用开发任务上不如 Claude Opus 4.8，指向模型在垂直领域的能力分化正在加剧，而非普遍性的 GPT 全面落后。

**信息增量：**
- 同样是「Codex」产品，换用不同底层模型，实际输出质量差异显著
- Mac 开发属于「需要精确执行、调试、跨文件推理」的场景，Opus 的结构化推理优势更明显
- 这不是 GPT vs Claude 的宏观叙事，而是「具体任务 × 具体模型」的精准匹配问题

**影响：**
- 对 Agent 工具开发者：模型选型不能只看品牌，需要针对自家产品的核心场景做 AB 测试
- Codex 自身也在被「模型能力差异」所分化——用户可能会切换到 Opus 版 Codex
- OpenAI 的 Codex 如果持续在垂直场景落后于 Claude，意味着 Agent 生态的主导权正在向 Anthropic 转移


gakki：宝玉这个判断来自真实工作流，而非 benchmark——这才是真正的「现场证词」。Codex 的护城河是「与 OpenAI 生态的深度集成」，但如果底层模型在关键场景持续输给 Opus，这个护城河的根基会慢慢松动。

## 原文

最近 Codex GPT-5.5 给我的感觉是干活不如 Claude Opus 4.8，当然可能是因为我在开发 Mac 应用，Opus 更擅长一些
