# Claude Managed Agents 记忆系统公测：Rakuten 首次出错率降 97% 背后的工程设计

## 摘要

**[Anthropic 为托管 Agent 上线文件型记忆系统，公测阶段开放]**

该记忆系统的核心设计思路是「实用主义」：以文件系统而非向量数据库存储记忆，Agent 可直接用 bash/代码操作读写，开发者可导出、回滚、设共享权限与审计日志。实测数据：Rakuten 团队通过记住历史错误将首次出错率降低 97%；Wisedocs 跨会话文档验证提速 30%；Netflix 智能体可将人工纠正自动带入下次会话。

共享存储支持多 Agent 并发访问而不互相覆盖，权限粒度从全公司只读到用户级别读写均可配置。

gakki：记忆系统的「文件优先」设计是正确工程直觉——让 Agent 用现有工具操作记忆，而非引入新的专用基础设施，这大幅降低了接入门槛。但 97% 出错率下降的前提是「同类错误重复犯」，这恰恰暴露了当前 Agent 在复杂任务中的浅层学习问题。

## 原文

Anthropic 今天为 Claude Managed Agents（托管智能体）上线了记忆功能，目前处于公测阶段。简单说，你部署的 AI 智能体现在能在多次会话之间积累经验，下次干活时不用从零开始。

这个记忆系统的设计思路挺实用主义的：记忆以文件形式存储在文件系统上，而不是搞一套专门的数据库或向量检索。好处是智能体可以直接用已有的 bash 和代码执行能力来读写记忆，开发者也能通过 API 导出、管理、甚至回滚到历史版本。

在权限设计上，记忆存储可以在多个智能体之间共享，并设置不同的访问范围。比如全公司级别的知识库设成只读，用户级别的存储允许读写。多个智能体可以并发访问同一个存储，不会互相覆盖。所有变更都有审计日志，能追溯到具体是哪个智能体、哪次会话产生的记忆。

从已经在用的团队来看，效果比较直观：Rakuten 的智能体通过记住过去犯过的错，把首次出错率降了 97%；Wisedocs 用跨会话记忆让文档验证流程提速 30%；Netflix 的智能体能把多轮对话中挖掘出的洞察和人工纠正带到下一次会话，不用再手动更新提示词。

对做智能体开发的团队来说，记忆一直是个痛点。大多数人要么自己搭一套 RAG 检索系统，要么每次会话手动塞上下文。Anthropic 把这层直接做进平台里，而且选了文件系统这个最朴素的方案，算是降低了不少工程复杂度。现在公测已经开放，可以在 Claude Console 里直接部署带记忆的智能体。
