# 中文 AI Skills 矩阵：从单点工具进化到「内容→发布」全链路流水线

## 摘要

**[中文 AI Skills 生态已从零散工具演化为覆盖内容生产→分发的完整套件]**

- guizang-social-card-skill：28 套版式 + 10 套主题，Monocle 电子杂志风 / 瑞士国际主义两套视觉系统，小红书 3:4 与公众号 21:9 一次生成
- baoyu-skills：宝玉 20+ 个 Skill 串成流水线——封面图→正文配图→Markdown 转公众号 HTML→推送公众号后台，公众号 API 接好即可发布
- 排版审美被直接打包成 Skill，把 Canva 设计师、公众号小编的排版工时打到接近零


gakki 锐评：中文 Skills 生态正在复刻 GPTs 当年的「数量爆发」路径，但分水岭已显现——单点 Skill 会被模型内化，能活下来的必须是一条工作流或多步骤编排。

## 原文

接上条，中文创作者开源的 AI Skill，再挑 5 个 这次偏排版和分发

从小红书配图、公众号排版，到 PPT、视频和长文写作，基本把「内容生产到发布」这条链路补齐了。

有几个确实做得很夸张：

6. guizang-social-card-skill
op7418 又一个，小红书卡片和公众号封面一条龙。
两套视觉系统，一套是 Monocle 那种电子杂志风，一套是瑞士国际主义。28 个版式、10 套主题，从小红书 3:4 组图到公众号 21:9 和 1:1 封面都能直接生成。

不需要打开 Canva，也不用自己一点点调字号和网格。

这哥们把排版审美直接打包成 Skill 了。
https://t.co/wtJWW8kZVW

7. baoyu-skills
宝玉的 20+ 个 Skill 套件，从内容创作到发布基本串成了一条流水线。

封面图生成 → 正文配图 → Markdown 转公众号 HTML → 推送到公众号后台。

配置好公众号 API 后，写完基本就能直接发。

做公众号最烦的往往不是写，而是找图、排版和发布。这套 Skill 解决的就是后半段。
https://t.co/f6HmTKYqJy

8. guizang-ppt-skill
把一篇长文直接提炼成 6—10 页的 HTML 横版翻页 PPT。两种视觉风格：电子杂志×电子墨水，以及瑞士国际主义。

支持键盘、滚轮和触屏翻页，生成的是单文件 HTML，可以直接发文件，也可以部署后分享链接。

看着像花了两小时排的，实际上只需要丢一篇文章进去。

op7418 是真把内容到视觉这个流程吃透了。
https://t.co/btq0qLKRV8

9. html-anything
nexu 团队出的，75 个 Skill 模板，覆盖 9 种内容形态。杂志文章、PPT、海报、小红书卡片、推文卡片、网页原型、数据报告和 HyperFrames 视频分镜都能做。

Markdown、Excel、CSV、JSON，甚至随手写的笔记，丢进去就能生成排版好的 HTML。

还支持一键导出公众号、X、知乎、HTML 和高清 PNG。

这个是今天几个里面最全的，但也最重。

适合那种「我什么都想做，但别让我学设计」的人。
https://t.co/4rOwI01tpC

10. content-research-writer
写长文的人应该会喜欢这个 它不是简单帮你生成一篇文章，而是从确定选题、整理大纲、搜索资料、添加引用，到优化开头和逐段修改，一步步陪你完成。

比较特别的是，它会先学习你的表达方式，再给出修改建议，不会直接把文章改成一股 AI 味。

适合写博客、教程、Newsletter、案例分析和需要引用资料的长文。
https://t.co/J6npffnvbi

以前做内容，需要在写作工具、Canva、PPT 和公众号后台之间来回切换。

现在这 5 个 Skill，已经可以把流程压缩成：搜集资料 → 写稿 → 配图 → 做 PPT → 排版 → 发布

AI 内容工具真正有价值的地方，不是替你多写几篇废话。

而是把那些重复、耗时，但又必须完成的工作接过去。
