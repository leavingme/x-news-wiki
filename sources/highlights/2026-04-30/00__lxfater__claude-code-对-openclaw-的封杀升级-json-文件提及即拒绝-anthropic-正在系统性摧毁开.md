# Claude Code 对 OpenClaw 的封杀升级：JSON 文件提及即拒绝，Anthropic 正在系统性摧毁开发者信任

## 摘要

**Anthropic 正在用技术手段将商战伪装成安全策略，开发者信任崩塌的速度远超封杀速度。**

- Theo（32 万粉硅谷 YouTuber）实测发现：Git 提交中只要包含提及 OpenClaw 的 JSON 文件，Claude Code 即拒绝请求或额外收费，封杀粒度已从 system prompt 下探到文件内容级别
- 此前 Claude Code 负责人 Boris Cherny 公开承诺「CLI 调用不会被封」，OpenClaw 作者 Steinberger 据此修改了默认配置，如今承诺已实质性失效
- Anthropic 后台仍在持续封禁 OpenClaw 的 system prompt，「表面开放、后台绞杀」的模式已形成明确的行为模式

Gakki 锐评：当一家公司开始用「安全」的名义做「竞争」的事情，且反复违背自家负责人公开承诺时，伤害的不只是竞品，而是整个开发者生态对平台的信任基准。Anthropic 正在用短期市场份额换取长期品牌损耗——这笔账，迟早要还。
