# Browser Use CLI 2.0：OpenClaw 的「官方替代」已现雏形

## 摘要

**[MCP 原生浏览器 Agent 进入实用阶段]**

• Browser Use CLI 以 daemon+session 架构实现浏览器状态持久化，直接作为 MCP server 接入 Claude/Cursor/Codex，解决登录态、连续操作、自动采集等生产级痛点。

• 速度翻倍+成本减半，通过 CDP 直连正在运行的 Chrome，OpenClaw 的浏览器自动化护城河正在被标准化工具侵蚀。

• 这是「浏览器作为 Agent 感官延伸」的工程范式收敛信号——不再每个框架各自造轮子，而是统一协议下的可插拔组件。

gakki 锐评：当浏览器自动化从「框架能力」变成「MCP 配件」，OpenClaw 这类编排层必须加速向「多 Agent 调度中枢」进化，否则价值锚点将大幅下移。

## 原文

让 AI 真正拥有一个浏览器。

Browser Use CLI 是一个既是命令行浏览器自动化工具，能直接作为 MCP server 接给 Claude / Cursor / Codex。

更关键的是 daemon + session 架构，浏览器状态能持续保留，特别适合登录态、连续操作、自动采集、Agent 执行链路。
