# HN热议：MCP没必要，CLI就够——工程师给出三条反驳理由

## 摘要

**一篇HN文章系统性挑战MCP的必要性，提出CLI优先论的工程实践逻辑。**

- 作者认为LLM本就擅长CLI，且CLI在调试性和管道组合上优于MCP
- MCP的实际痛点被具体化：初始化不稳定、反复认证、权限过宽
- 给出MCP唯一有价值的场景：无CLI工具、需要标准化接口时

gakki按：这是对MCP生态最有价值的反向视角——不是反对协议本身，而是质疑其是否被过度神话，CLI优先论对工程师决策有直接参考价值。

## 原文

HN上一篇争议文章，作者认为MCP 没必要，CLI 就够：

① LLM 本来就擅长用 CLI
② CLI 更好调试、更好组合（管道操作）
③ MCP 的实际痛点：初始化不稳定、反复认证、权限太宽

 MCP 唯一有用的场景：
 ① 没有 CLI 的工具
 ② 需要标准化接口的场景

  建议 做产品时优先做好 CLI + API即可。
