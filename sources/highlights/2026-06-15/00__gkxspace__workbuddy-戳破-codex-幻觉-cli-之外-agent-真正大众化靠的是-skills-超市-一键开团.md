# WorkBuddy 戳破 Codex 幻觉：CLI 之外，Agent 真正大众化靠的是「Skills 超市 + 一键开团」

## 摘要

**企业 Agent 产品化正在绕过 Codex 式 CLI，用 Skills 商店 + 多专家编排吃掉「不会写命令行的普通用户」这块最大增量市场。**

- 余温把 WorkBuddy 定位为「Codex 门槛太高」的替代品：内置数据分析师、PPT 设计师、小红书运营、法务等上百个 AI 专家，配套 Skills 商店、自动化、远程遥控
- 核心交付形态是「说一句需求 → 自动拆解 → 多专家并行 → 整合交付」，已用「做 AI 编程工具竞品分析报告+PPT」「7 份 PDF 提取对比表」「每日早 8 点 AI 行业日报」等场景验证
- 零命令行、零配环境、免费起步——这恰好对应了 no-code Agent 入口这条已被验证的产品化路径
- 关键反差：Codex/Claude Code 越往开发者深水区走，WorkBuddy 这类产品越在抢占「不敢开终端的运营/产品/老板」这个 10× 更大的池子


gakki 锐评：这是企业 Agent 产品化「去极客化」的典型样本——价值不在底层模型谁强，而在谁能把 200 个 Skill 装进一个开箱即用的壳子。但「专家能组团」目前还是产品话术，真正决定生死的是三件事：专家 Skills 的真实召回率、跨专家交付物的整合质量、以及企业付费场景里能否接住数据合规。

## 原文

如果要推荐一个“普通人也能用起来的 AI Agent”，我会推 WorkBuddy，因为 Codex 对大众终究是门槛太高了。​

WorkBuddy 最近更新很快，而且还内置了一堆专家智能体、Skills 商店、自动化、远程遥控...，其实对于大多数人完全够了，里面各种封装好的技能让小白也能轻易上手。
​
WorkBuddy 里面上百个 AI 专家：数据分析师、PPT 设计师、小红书运营专家、竞品研究员、法务顾问、文案策划、翻译...​

而且这些专家可以组团。你说一句需求，他自动拆解任务分配给不同专家并行执行，最后整合成一份完整的交付物。​

比如我说"帮我做一份 AI 编程工具竞品分析报告加 PPT"：​
一个专家去搜资料​
一个在整理数据对比​
一个在排版 PPT​
最后做整合交付​
​
当然实际用法太多了：​
“帮我把上这 7 份 PDF 报告读完，提取每篇核心观点做成对比表格”、“每天早上 8 点自动搜 AI 行业热点，整理成日报存到桌面”、“围绕 AI 效率工具帮我出 3 个小红书选题，选最好的写完整图文”​
​
下载很容易，也不用配环境、不用命令行，并且免费开始，也允许通过 API 密钥接入任意大模型。​
​
