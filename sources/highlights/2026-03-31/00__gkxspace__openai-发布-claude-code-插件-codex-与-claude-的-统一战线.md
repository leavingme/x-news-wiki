# OpenAI 发布 Claude Code 插件：Codex 与 Claude 的「统一战线」

## 摘要

**[OpenAI 主动打破平台壁垒，将 Codex 嵌入竞争对手 Claude Code 的生态]**

• 战略转向：OpenAI 不再坚持「二选一」，而是让 Codex 成为 Claude Code 的插件，提供代码审查、对抗性审查及任务移交能力
• 信号解读：这既是 OpenAI 对自身工具链信心的体现，也暗示 Agent 工具市场进入「互操作性优先」阶段
• 影响边界：开发者不再需要在 Claude Code 和 Codex 之间做选择，而是可以按需调用，降低切换成本

**gakki 锐评**：OpenAI 这一招看似「大度」，实则是把 Codex 定位为「基础设施层」——与其在入口层和 Claude Code 硬碰硬，不如成为所有 Agent 的「底层引擎」。这是平台战争的升维打法。

## 原文

OpenAI 牛逼啊，格局太大了。

这下不用选择是用 ClaudeCode 还是Codex了，直接在 ClaudeCode 里面用 Codex。

OpenAI 发布的 Claude Code 插件 codex-plugin-cc，能直接在 Claude Code 里调用 Codex 做代码审查、对抗性审查，甚至把任务整个移交给 Codex 执行。
