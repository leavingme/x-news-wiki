# OpenAI 发布「一键迁移到 Codex」：从 Claude Code/Cursor 抢人的最激进动作

## 摘要

**OpenAI 直接把竞品用户迁移做成了一键功能，这是平台战争进入存量争夺阶段的标志。**

- 支持从 Claude Code、Cursor 导入配置、规则、技能、MCP、hooks、subagents 及近 30 天会话
- instruction files → AGENTS.md，settings.json → config.toml，自动映射后剩余部分由 Agent 线程接管
- 这不是技术功能，是用户心智争夺——降低切换成本等于降低忠诚度壁垒

gakki 锐评：当平台开始帮你搬家，说明你已经是它们要抢的资产。Codex 的真正对手不是 Claude Code 的功能，而是用户的习惯惯性。

## 原文

釜底抽薪

OpenAI 搞了个 Migrate to Codex 功能

让你可以把其他编程工具，比如Claude Code、Cursor里里的配置，一键导入到 Codex

包括编程 Agent 里的配置、规则、技能、MCP、hooks、subagents、最近30天的所有会话等...

它会扫描用户级配置和项目级配置，把能自动映射的内容导入 Codex。比如 instruction files 会迁移到 AGENTS.md，settings.json 会迁移到 config.toml，MCP、hooks、skills、subagents 也都有对应入口。

更有意思的是，不能自动迁移的部分，Codex 会直接开一个新线程，让 Agent 继续帮你处理剩余迁移。
