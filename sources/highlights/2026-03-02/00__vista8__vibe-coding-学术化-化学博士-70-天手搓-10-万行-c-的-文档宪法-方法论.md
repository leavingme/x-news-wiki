# Vibe Coding 学术化：化学博士 70 天手搓 10 万行 C# 的「文档宪法」方法论

## 摘要

**Vibe Coding 的隐性前提被戳破：不是不写文档，是用不同粒度写。**

- 按人类「短期/工作/长期记忆」分层构建文档，将 AI 可读性纳入设计
- 「解释过两次就写下来」——重复解释是知识固化的信号
- 专家 Agent 作为「调试重启点」，卡顿时重建上下文比硬磕更高效

化学背景作者写分布式系统，这本身就是 Vibe Coding 的隐喻：领域边界正在消融，表达能力成为新硬通货。

## 原文

没想到还有论文讲Vibe Coding技巧的，作者是化学专业。

70天，写了一个10 万行的 C# 分布式系统。

比较值得参考的就是文档化，按人类的短期记忆、工作记忆和长期记忆来划分构建文档。

几条建议：
1.尽早建“宪法”
哪怕只写项目目标、技术栈、核心约定，也能从第一天起避免一大堆 AI 错误。 

2.写给 AI 看，不是给人看
文件路径、函数名、明确的"做这个/别做那个"指令。 

3.解释过两次，就写下来
重复解释同一个领域知识，就是该固化成文档的信号。 

4.卡住时，创建专家 Agent 重启
调试某个领域卡了很久？与其继续磕，不如花时间建个专家 Agent，重启任务。
