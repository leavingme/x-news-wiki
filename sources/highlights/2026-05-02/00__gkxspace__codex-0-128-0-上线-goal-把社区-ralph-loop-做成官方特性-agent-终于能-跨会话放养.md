# Codex 0.128.0 上线 /goal：把社区 Ralph Loop 做成官方特性，Agent 终于能「跨会话放养」

## 摘要

**[Codex 将社区最佳实践「Ralph Loop」官方化，Agent 自主执行的工程范式正式落地]**

- 新增 /goal 功能：给 Codex 一个长期目标，它会跨多回合持续规划→执行→验证→迭代，直到达成或触达 token 预算上限
- 配置极简：`codex features enable goals` 或 config.toml 一行开关，零摩擦启用
- 适用场景：项目重构、框架迁移、多模块 bug 修复——以前需要人盯好几轮的任务，现在可以真正放养

gakki 的判断：这是 Agent 从「对话式助手」向「自主工程伙伴」跃迁的关键一步。把社区验证过的 Ralph Loop 收编为官方特性，说明 OpenAI 终于承认：最好的 Agent 交互不是逐句指挥，而是给方向、放手跑。但 token 预算上限这个约束很诚实——自主性的边界仍然由算力成本决定。

## 原文

终于！Codex 0.128.0 把大家盼了好久的 /goal 功能正式上线了！

这意味着不用一句一句指挥 AI 干活。

这个功能本质是把社区里流传的 Ralph loop 做成了官方特性，你给它一个长期目标，它会把目标“记心里”，跨多回合持续工作，不停规划、执行、验证、迭代，直到达成（或者达到你设的 token 预算上限。。。）

配置超级简单：
在 ~/.codex/config.toml 最后加上
[features]
goals = true

或者，告诉Codex：
codex features enable goals

比如项目重构、框架迁移、多模块 bug 修复这些以前要盯好几轮的任务，现在可以真正放养了。

Codex 真是越来越好了，它不再只是个聪明的代码助手，正变成一个有持久力、有自主性的 AI 工程伙伴。

如果你也在用 Codex CLI，强烈建议赶紧更新试试～
