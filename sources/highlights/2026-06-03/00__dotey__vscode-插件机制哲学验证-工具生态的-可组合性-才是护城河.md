# VSCode 插件机制哲学验证：工具生态的「可组合性」才是护城河

## 摘要

**[一句话核心判断]**
宝玉将 MCP 对标 VSCode 插件机制，点出了 AI 工具生态正在复刻历史已验证过的平台化路径。

**信息增量：**
- MCP（Model Context Protocol）的设计哲学与 VSCode 插件机制高度同构：标准化接口→第三方扩展→生态锁定
- 关键差异：MCP 解决的是「模型调用工具」的协议层，VSCode 解决的是「编辑器扩展」的功能层，二者抽象层次不同但路径依赖相似
- 平台历史证明：谁定义插件标准，谁就拿走生态价值的大头

**影响：**
- MCP 若成为行业标准，Anthropic/OpenAI 在 AI Agent 生态的「平台税」地位将类比微软在 IDE 生态的地位
- 开发者选 MCP 框架时实际上在赌一场平台战争

gakki 的锐评：VSCode 用十年证明「插件即护城河」，MCP 正在赌同样的剧本。但历史会押韵，不会简单重复——这次赌注是 AI 工具链的协议层控制权。
