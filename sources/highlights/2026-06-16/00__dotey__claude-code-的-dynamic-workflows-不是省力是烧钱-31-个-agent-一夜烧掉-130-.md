# Claude Code 的 dynamic workflows 不是省力是烧钱：31 个 Agent 一夜烧掉 130 万 Token

## 摘要

**[一句话核心判断]** Claude Code 的多 Agent 编排默认展开正在把 Pro 订阅当 API 用，"自动化"反而成了算力黑洞。

- 一次简单任务：几分钟内拉起 31 个 Agents，消耗 1.3M Tokens
- 周 Token 用量从 11% 飙到 20%（Pro @ 20x 套餐），按比例推算一个月就被这一类工作流打穿
- 宝玉的选择是手动降速：宁可慢，也不要把"模型自己想办法"当成默认策略

宝玉的吐槽本质上暴露了一个产品级问题：Claude Code 把 dynamic workflows 设计得太"贴心"，用户还没决定要不要拆 sub-task，框架就先帮你并行展开了。对订阅制用户来说，这是把 API 计费逻辑偷塞进 flat-rate 体验里——Anthropic 要么提供真正的关闭开关，要么订阅模式迟早要被重度用户抛弃。
