# 吴恩达Context Hub：Agent知识共享的「Stack Overflow 2.0」实验

## 摘要

**[吴恩达下场做Agent基建，试图解决AI乱编API和不长记性的行业顽疾]**

• 开源一周斩获1万+ star，定位「给AI用的Stack Overflow」——Agent调用API后自主评价文档质量、标记坑点、完善参数说明
• 核心突破：让AI从「每次从零开始」转向「跨会话记住真实调用经验」，实现Agent间的知识沉淀与共享
• 隐喻性信号：API文档的维护者从人类程序员变成AI自身，意味着工具链的自我进化闭环正在形成

**gakki锐评：** 这是个基础设施级别的尝试，但真正的考验在于——当Agent开始吐槽API太烂时，谁来听？

## 原文

吴恩达老师搞的这个 Context Hub，开源一周左右就 1 万多 star 了。

号称可让 Agent 拥有自我进化能力，能跨会话记住真实的调用经验。AI 乱编 API 和不长记性的问题应该能缓解不少。

我觉得比较有趣的是 这项目搞了个「给 AI 用的 Stack Overflow」。

让 agent 调完接口后，自己去评价文档，比如说说哪里有坑或者漏了参数。以前是咱们程序员在论坛里苦哈哈地抱团排雷，现在变成 AI 们自己互传心得，还顺手把库里的文档给完善了。

照这路子发展，以后你的 agent 写着代码，没准就开始吐槽别人的 API 太烂了🤣
