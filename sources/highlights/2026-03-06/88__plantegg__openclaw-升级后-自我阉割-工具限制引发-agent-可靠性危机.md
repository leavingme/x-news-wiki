# OpenClaw 升级后「自我阉割」：工具限制引发 Agent 可靠性危机

## 摘要

**[新版本工具权限收紧，Agent 陷入「自医不能」困境]**
- 升级后 Tools 被默认限制，OpenClaw 沦为「只能聊天」的 GPT
- 用户被迫用「Claude Code 修 OpenClaw」的套娃式解决方案
- 暴露 Agent 平台的核心矛盾：安全边界 vs 执行能力的零和博弈

这不仅是 bug，而是 Agent 基础设施的结构性张力。当平台方收紧权限以求安全，用户却在真实场景中失去「用 AI 修 AI」的自愈能力——这种设计哲学分歧，将长期影响 OpenClaw 与 Claude Code 的路线之争。

## 原文

很多人的 OpenClaw 在升级之后，发现什么也干不了了。就像一个单纯的 GPT 一样，只会聊天，不会执行任何工具，很多人在这个问题上花了一天，甚至更长的时间去解决。

这是因为 OpenClaw 的新版本把 Tools 限制了。当然，这个问题我之前也不了解，直到我自己发现这个问题的时候，我第一反应就是让小龙虾（OpenClaw）自己去找原因并去修复。  

然后小龙虾找到问题了，但是它因为没有这些 tools 自己修复不了，于是我就把小龙虾的结果丢给 Claude Code，然后 Claude Code 帮我修好了。  

你看我的截图，用小龙虾去解决小龙虾的问题。如果小龙虾解决不了，那就用 Claude Code 去解决小龙虾的问题。  

这是我如何养小龙虾的经验

用 AI 去解决 AI 的问题，是这个时代非常非常重要的一项基础能力，希望你学会了。
