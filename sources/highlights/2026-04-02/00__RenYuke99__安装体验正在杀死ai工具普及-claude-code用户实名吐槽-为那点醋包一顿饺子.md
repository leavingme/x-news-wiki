# 安装体验正在杀死AI工具普及：Claude Code用户实名吐槽「为那点醋包一顿饺子」

## 摘要

**一句话核心判断：AI工具的「最后一公里」不是模型能力，而是安装配置的摩擦成本——这个认知鸿沟正在系统性地排斥80%的潜在用户。**

→ 用户实测：即使具备Linux运维背景，安装Claude Code和OpenClaw仍需跨越账号、网络、配置三重大山
→ 「用户只是想去体验试着去学习，但前期投入成本就劝退80%以上的人」——这是典型的「技术傲慢」产品设计
→ 核心矛盾：工具设计逻辑仍以「工程师友好」为导向，而非「零基础用户即开即用」
→ OpenClaw的使命应该是「让工具服务人，而不是让人去学习使用工具」——但当前版本显然还没做到

gakki锐评： 这个吐槽本质上是Vibe Coding最大的讽刺——当你需要花3小时配置环境才能「vibe」起来，那个vibe早就死了。产品化能力是AI工具的生死线。

## 原文

一个好的AI产品应该做到0基础直接上手，开箱即用

现在想做vibe coding还是太难，去装一个coding软件就要费好大功夫、、

今天装了Claude code & openclaw 2026.4.1
说一下我的感受
恶心，太他妈恶心了
虽然我不算资深程序员，但也学过Linux运维之类的课程，还是懂点代码的，我仍然觉得安装起来有点困难

解决了技术问题你还要考虑账号，网络，以及各种配置，这些对于小白用户还是太难了，用户只是想去体验试着去学习，但这些工具前期的安装以及买账号买网络的投入成本就劝退80%以上的人了

体验下来就是为了那点醋去包一顿饺子

大模型提供商以及中转站服务商必须要把用户痛点解决掉才能把蛋糕做大，去让更多的用户消耗token，才能创造更多的收入
 @openclawby 这个项目就是要为解决用户使用安装困难而开发，让工具服务人，而不是让人去学习使用工具！
