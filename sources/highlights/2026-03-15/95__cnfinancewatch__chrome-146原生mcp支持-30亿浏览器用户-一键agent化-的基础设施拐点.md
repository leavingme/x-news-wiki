# Chrome 146原生MCP支持：30亿浏览器用户「一键Agent化」的基础设施拐点

## 摘要

**浏览器Agent化的「iPhone时刻」正式到来。**

• Chrome 146在flags中内置WebMCP测试开关，意味着AI Agent操控浏览器从「第三方hack」变为「官方原生能力」
• 此前需要Puppeteer脚本或Lightpanda等外挂方案，现在用户只需启用flag即可让CLI Agent直接连接浏览器
• 这对Agent工具链意味着：浏览器感知层即将标准化，Token效率与执行可靠性将迎来数量级提升

Chrome不再是「被操控的客体」，而正在成为Agent的「原生感官器官」。这对于OpenClaw等CLI Agent生态是重大利好——浏览器自动化终于要从「脆弱的黑魔法」变成「稳定的公共基础设施」。

## 原文

开启步骤：
1/ 升级到 Chrome 146
2/ 打开 chrome://flags/#enable-webmcp-testing，设为 Enabled
3/ 重启 Chrome → 你的 CLI agent（比如 OpenClaw）就能通过 MCP 协议连上去了。

以前 AI agent 操控浏览器要靠第三方工具或者自己写 Puppeteer 脚本，现在 Chrome 官方把这个能力内置了。全球 30 亿 Chrome 用户，理论上都可以一键把自己的浏览器变成 AI agent 的操作界面。
