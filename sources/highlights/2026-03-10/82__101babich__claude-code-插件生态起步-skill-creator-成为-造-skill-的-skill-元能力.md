# Claude Code 插件生态起步：Skill Creator 成为「造 Skill 的 Skill」元能力

## 摘要

**[Claude Code 进入「插件化」扩展阶段]**
- Skill Creator 作为元插件，实现「描述→生成→测试→迭代」的 Skill 自动化生产
- Figma 等工具接入，说明 Claude Code 正向「设计-开发一体化」渗透
- 插件生态是 Claude Code 从「个人效率工具」向「团队协作基础设施」跃迁的关键信号

当 Claude Code 长出插件系统，它就不再是一个编辑器插件，而是一个「可被无限扩展的 Agent 操作系统」。Skill Creator 的价值在于把 Skill 开发门槛从「写代码」降到「写描述」。

## 原文

🔥 Top 7 Claude Code Plugins

Claude Plugins are nice ways to extend what Claude can do.

I want to share my top 7 favorite plugins:

1️⃣ Skill Creator

Skill Creator should be the first plugin you install to Claude. Despite it being called a plugin, it’s actually a meta-skill that allows you to create better skills based on a description you provide. It drafts a new skill for you, evaluates its performance, and refines a new skill based on the results of testing.

2️⃣ Figma

This plugin allows you to have a two-way connection between Figma and Claude Code. It means that you can design something in Figma and turn it into code, or build something in Claude Code, transfer it to Figma (to refine it there), and then send it back to Claude to code the change! This makes the process of iterating on design less painful and more visual.

3️⃣ frontend-design

Front-end design is one of the most popular use-cases for Claude Code. No wonder that many product people (including myself) create their own front-end design skills/plugins to enrich Claude’s ability to craft unique designs. But the great news is that you can skip the step of manual skill/plugin creation, simply use frontend-design skill. Instead of generating generic AI layouts, this plugin helps Claude produce distinctive UI with strong visual design and clean code.

4️⃣ context7

One of the typical problems with Claude Code happens when AI uses an incorrect version of the code framework and crafts the code that doesn’t work with your codebase. context7 plugin solves this problem by giving Claude version-specific documentation and code examples from official repositories. This information is injected into Claude’s context, so you will always use up-to-date technical documentation directly inside Claude when coding.
