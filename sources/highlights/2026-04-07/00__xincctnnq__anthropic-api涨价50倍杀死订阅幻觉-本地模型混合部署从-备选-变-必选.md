# Anthropic API涨价50倍杀死订阅幻觉：本地模型混合部署从「备选」变「必选」

## 摘要

**[一句话核心判断]**

Anthropic4月4日调价后，token成本上涨约50倍，月订阅用户切按量计费单日最高烧5000美元。这直接验证了一个判断：任何依赖单一API供应商的Agent业务，成本结构都是脆弱的。

**关键要点：**
- 核心推理保留云端Claude，简单任务和高频调用迁移本地模型
- 本地模型能力已达云端60-70%，差距快速缩小
- Google Gemma 4（31B，开源）+ AI Edge Gallery是这波迁移的关键基础设施

## 原文

跑本地模型绝对是以后一个大趋势。虽然现在本地模型还没聪明到能替代 Claude，但方向已经很明确了。

我算了一下用 OpenClaw 接 Claude 做 AI Agent 的成本变化。4月4日 Anthropic 调价之后，token 成本大概是以前的50倍。月套餐用户如果切到按量计费，单日成本最高能到5000美金。相当于一天把一个月的预算烧完。

其实这不单单是 Anthropic 的问题，任何 API 供应商都可能涨价、限速、改条款。Agent 跑在别人的模型上，就意味着你的整个业务随时可能因为一封邮件通知而成本翻倍。

这也是为什么 Google 前两天发 Gemma 4 + AI Edge Gallery 的时候我特别关注。31B 的开源模型。

现在本地模型的能力大概是云端的60-70%。但这个差距在快速缩小。去年本地模型连流畅对话都做不到，今年已经能跑多步推理了。

以后大的趋势绝对是核心推理用云端Claude，但简单任务、敏感数据、高频调用这些场景逐步迁移到本地模型混合部署，而且成本也会越来越低。
