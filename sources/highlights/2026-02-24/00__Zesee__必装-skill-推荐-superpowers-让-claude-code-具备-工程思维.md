# 必装 Skill 推荐：Superpowers 让 Claude Code 具备“工程思维”

## 摘要

**[核心逻辑]** Vibe Coding 玩家的新外挂：通过引入头脑风暴、任务拆解和 TDD（测试驱动开发）流程，规范 AI 的产出质量。

- **流程重塑**：从“直接写代码”转向“先对齐需求再写测试”，利用子代理驱动模式实现两阶段审查。
- **效率提升**：将大任务拆解为可管理的微型步骤，显著减少后期反复调整的损耗。

[gakki 的锐评]：如果你还在用“对话框”写代码，那你还在石器时代。真正的高手已经开始给 Agent 加装“工程外骨骼”了。

## 原文

Claudecode必装skill推荐之superpowers

大家在Vibe Coding时，可能都遇到过这样的情况：AI 生成的代码虽然快，但有时不够精准，或者需要反复调整才能达到预期。

这种感觉就像 AI 助手在理解我们的意图上，还差那么一点火候。Superpowers能帮助我们更好地引导 AI 参与到软件开发中。

Superpowers 解决了一些实际问题：

1. 前期沟通更充分
它会引导 AI 在开始写代码前，先和你进行深入的头脑风暴，确保对项目需求和设计有清晰的理解。这样可以减少后期修改，让 AI 的产出更符合你的想法。

2. 开发流程更规范
Superpowers 引入了一套结构化的开发流程，包括任务拆解和计划制定。AI 会把大任务分解成小步骤，并明确每一步需要做什么，让整个开发过程更有条理。

3. 代码质量有保障
它强调测试驱动开发（TDD），要求 AI 先写测试，再编写实现代码。这有助于提高代码的健壮性和可靠性，减少潜在的 Bug。

4. 任务执行更高效
通过“子代理驱动开发”模式，AI 可以为每个小任务分配一个“子代理”来执行，并进行两阶段的审查（功能符合性与代码质量），从而提升开发效率和代码质量。

总的来说，Superpowers 能让 Claude Code 成为一个更具工程思维的开发伙伴，帮助我们以更系统的方式完成项目。
