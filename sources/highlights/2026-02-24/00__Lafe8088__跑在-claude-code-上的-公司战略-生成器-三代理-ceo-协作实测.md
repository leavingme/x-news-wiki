# 跑在 Claude Code 上的“公司战略”生成器：三代理 CEO 协作实测

## 摘要

**[核心逻辑]** 创业者利用 CEO、CPO、CMO 三个 AI 代理在终端内协作，一条命令生成竞品分析与战略文档。

- **迭代思考**：AI 方案不一定全对，但输出的过程强迫人类建立判断标准，通过 Commit 记录每一次决策的 diff。
- **从框架到决策**：利用 AI 提供“粗糙的框架”来对冲盲目决策的风险。

[gakki 的锐评]：比起拍脑袋，AI 拍出来的方案至少有逻辑。最重要的不是方案，是它逼你思考那些以前不敢碰的盲点。

## 原文

一条命令，生成你整个公司的战略
开源了，跑在 Claude Code 上。 https://t.co/p7KV1GO9rP

创业融了几千万之后我发现，最难的不是没钱，是想不清楚该做什么。

每天一堆没有标准答案的决策。做 B 端还是 C 端？先切哪个行业？功能做广还是做深？渠道投哪个？全堆在脑子里，根本想不清楚。人的带宽是有限的，什么都想，等于什么都没想。

后来我搞了个东西。三个 AI 代理，CEO、CPO、CMO，跑在终端里。一条命令，它帮你把目标拆解、竞品分析、渠道评估、版本规划全过一遍，输出一份完整的战略文档。

说实话 AI 做出来的方案不一定都对。有时候竞品分析会漏东西，渠道评分权重也不一定合理。

但这不是重点。

重点是，你打开它的输出，会被迫去想一些以前从来没想过的问题。「这个评分合理吗？」「这条渠道为什么分低？」「我自己会怎么打？」你开始产生自己的判断标准。跑第二遍的时候调整权重，第三遍更准，第四遍更准。

不是 AI 越来越聪明了。是你越来越清楚自己要什么了。

而且所有输出都是 markdown，全在 Git 里。每一次决策都有记录，能回溯，能 diff。三个月后复盘不用靠记忆，翻 commit 就行。

有框架总比拍脑袋强。哪怕框架一开始很粗糙。
