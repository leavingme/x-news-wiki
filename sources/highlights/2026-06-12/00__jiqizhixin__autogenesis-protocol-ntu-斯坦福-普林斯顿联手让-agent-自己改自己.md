# Autogenesis Protocol：NTU/斯坦福/普林斯顿联手让 Agent 自己改自己

## 摘要

**自进化 Agent 从论文走向工程协议：版本管理、升级提案、安全回滚首次被标准化。**
- NTU、Stanford、Princeton 等机构联合发布 AGP（Autogenesis Protocol），将 Agent 的"使用层"（prompt、工具、记忆）与"改进层"解耦。
- 配套系统 AGS 在复杂长程规划与工具调用基准上稳定击败强基线，标志着 Self-Improving Agent 从概念走向可复现的工程实践。
- 回滚机制是亮点——意味着 Agent 自我迭代不再是"黑盒进化"，而是带版本控制的工程流程。


gakki 锐评：自进化一直是 Agent 圈的最大叙事但落地最差，AGP 的价值在于把"自我修改"做成了有 Git 思维的东西——但能否在生产 Agent 里跑通，还要看回滚策略的颗粒度。

## 原文

What if AI agents could evolve themselves to get better over time?

Researchers from NTU, Stanford, Princeton, and others introduce Autogenesis Protocol (AGP). 

It separates what agents use (prompts, tools, memory) from how they improve—letting agents track versions, propose upgrades, and roll back changes safely.

The result: their Autogenesis System (AGS) consistently beats strong baselines on complex, long-horizon planning and tool-use benchmarks. Self-improving agents are here.
