# 余温实测：Codex 插件是 Agent 时代的 App Store，不会用等于把 iPhone 拿来只打电话

## 摘要

**[Codex 真正的护城河不在模型，而在插件矩阵——这才是 Agent 平台战的胜负手]**

- Chrome 插件 + Computer Use 接管所有网页与桌面操作，"一句话甩过去"已成默认动作
- Vercel/Supabase 等部署插件让小白在对话框里完成上线，把 Agent 工具链压缩成"一站式对话流"
- 自动化测试、配置排查全部外抛，"懒的状态回不去"是 Agent 工具真正的用户黏性来源


gakki 锐评：模型差距在收敛，工具生态差距在放大。Codex 押插件矩阵、Claude Code 押 Skill——殊途同归，都是把"操作界面"内化进 Agent 对话框，谁先把覆盖面卷满，谁就吃掉下一波用户。

## 原文

不会用 Codex 插件，相当于 iPhone 只拿来打电话……

这份 Codex 插件清单，建议直接收藏，但有一点我跟他正好相反：
Chrome 插件我天天用，它和 Computer Use 是我用得最爽的两个，现在各种任务我基本都直接甩给它们。

网页上的各种操作，懒得自己点（或者说找不到，hhh），一句话丢过去得了
电脑上的配置问题，让它自己开设置自己排查
一些自动化测试，也直接让 AI 自己跑、看结果、自己改

另外 Codex 插件里的好东西还有不少，Vercel、Supabase 这些我也一直在用，小白也能部署上线都不用离开对话框

人一旦习惯了这种懒的状态，是真回不去了
