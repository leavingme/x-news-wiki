# xAI 正式入局 AI Coding 工具战：Grok Build 开放测试，对标 Claude Code/Codex

## 摘要

**【一句话核心判断】**

Grok Build 开启测试，xAI 正式将 CLI 编程工具纳入产品矩阵，AI Coding 赛道从双头垄断（Anthropic + OpenAI）进入三足鼎立阶段。

**信息增量：**
- 覆盖用户：SuperGrok + X Premium+ 订阅用户，门槛明确（订阅制 vs OpenAI 的免费+付费混合）
- 核心能力对齐：Plan 模式、Always-approve 子 Agent 并行、@文件索引、MCP 扩展（Linear/Sentry/Postgres/浏览器）、Skills 自定义技能
- 会话管理：/new /load /rename /share /context /compact，完整生命周期覆盖
- 安装极简：curl 一键安装，对非技术用户门槛极低

**影响分析：**
- MCP + Skills 双扩展机制，等于同时对齐 Anthropic 的 Skills 生态和 OpenAI 的工具调用范式
- X 平台分发优势：SuperGrok 用户可直接授权，获客成本趋近于零
- 子 Agent 并行是差异化亮点，Claude Code 尚未原生支持此能力

Grok Build 是 xAI 补全"从聊天到干活"闭环的关键一步。订阅门槛和 X 平台分发是竞争优势，但真正的考验是生产环境稳定性——Grok 模型在代码补全和debug场景下的准确率是否经得住验证，这才是它能从内测进入主流开发工作流的核心指标。

## 原文

Xai的 CLI：Grok Build 

开启测试了 

对标 Claude Code、Codex

SuperGrok 和 X Premium+ 用户可以直接安装并授权账户即可使用，使用订阅的用量

安装：curl -fsSL https://t.co/Z30H5ruX17 | bash

启动：grok

Grok Build 基本功能清单：

核心模式

Plan 模式：先出方案、不直接改文件，可停下来反问
Always-approve 模式：跳过每次工具调用的确认提示
Shift+Tab 切换会话模式

编程能力

读代码、写代码、改代码、修 bug、跑测试
子 Agent 并行：拆分大任务，多个子 Agent 同时干
用 @文件名 直接定位某个文件讨论

会话管理

/new 开新会话、/load 续上旧会话、/rename 改名
/share 生成链接分享会话
/context 查看上下文用量、/compact 压缩历史
/btw 不打断当前任务，随口问个别的

扩展能力

MCP：接 Linear、Sentry、Postgres、浏览器等外部工具
Skills：自定义技能，装上后变成斜杠命令调用
Plugins、Hooks 支持

多模态

/imagine 文字生成图片
/imagine-video 文字生成视频
