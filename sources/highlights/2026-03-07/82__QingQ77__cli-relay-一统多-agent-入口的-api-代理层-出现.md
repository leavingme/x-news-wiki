# CLI Relay：一统多 Agent 入口的「API 代理层」出现

## 摘要

**【Agent 工具整合进入「中间件」阶段】**

- 打破各 AI CLI 工具（Claude Code、Gemini CLI、Codex、Amp）的接入孤岛，统一本地 API 入口
- 支持 OAuth 与 API Key 双模式，可复用现有账号体系与订阅配额
- 为多账号、高并发调用场景提供工程化解决方案，降低 Agent 基础设施的碎片化成本

这种「统一代理层」的出现，标志着 Agent 工具从各自为战的蛮荒期，进入可编排、可治理的整合期。开发者不再被迫在多个 CLI 生态间反复横跳，而是可以通过一层抽象统一管理调用策略、权限与配额。

## 原文

CLI Proxy API 项目的魔改版

CliRelay  一个把多个 AI CLI 工具统一接到本地入口的代理服务器。

它可以让 Claude Code、Gemini CLI、OpenAI Codex、Amp CLI这类工具共用同一个本地 API，不用每个客户端单独配一套接入方式。它同时支持 OAuth 和 API Key，适合直接复用你现有的账号、订阅和密钥体系。

如果你有多个账号、多个项目，或者调用频率比较高，它也能派上用场。
