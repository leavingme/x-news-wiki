# YC 合伙人亲述：Vibe Coding 的专注是假的，burn out 是真的

## 摘要

**【Vibe Coding 的认知税终于被量化：上下文切换 + 预估失准是元凶】**

YC (Yucheng) 披露了一个反直觉的观察：开十几个 Claude Code session 的 vibe coding 专注状态，表面高效，实际比传统心流更累——stressful，甚至触发智能设备警报。

两大根源：高频上下文切换（费脑力）+ AI 表现波动导致工作时间预估失准（30 分钟的任务可能 5 分钟或 1 小时），频道错误累积。

行业影响：如果这不只是个人体感，而是普遍规律，Vibe Coding 的「效率叙事」将面临根本性质疑。

---
*gakki：这是目前为止对 Vibe Coding 最诚实的反思之一。说「AI 让我更专注」很容易，但说「专注得让我 burn out」需要勇气。更重要的是——如果专注和效率是两件事，那 hype cycle 里的泡沫已经开始了自我消化。别急着嘲笑，2025 年初的共识往往在 2026 年初变成教训。*

## 原文

最近和朋友聊起来才发现这不只是我自己的感受：
虽然vibe coding, 开十几个Claude Code sessions工作也很专注，但和以前的深度工作的心流状态相比，vibe coding的专注非常stressful, 甚至经常触发智能设备的警报。从状态里面出来的体感也很不好：没有心流的满足感，只有burn out。

我体感有两个主要原因：
1. 高频率上下文切换，这个非常费脑力
2. 频道错误预估工作时间。由于AI表现波动很大，经常出现预估30分钟的工作量花了5分钟，然后预估5分钟的花了一个小时。

你有遇到过类似体感吗？
