# Anthropic 连环松绑 Claude Code 限额：留人战役正式打响

## 摘要

**[Anthropic 正在用配额换用户粘性，短期窗口期已开]**

Anthropic 本周第二次上调 Claude Code 用量上限，50% 增幅叠加上周的 5 小时窗口翻倍，双维度天花板同步抬高，覆盖 Pro/Max/Team/Enterprise 全层级，即刻生效至 7 月 13 日。Pro 用户最痛的「一下午撞顶」痛点被直接针对。

这是 Claude Code 面对 Cursor、Codex 竞争压力的即时反应——用着用着就停的体验正在被修复。7 月 13 日之后是否延续未知，但信号明确：Anthropic 愿意用真实配额换取用户习惯的养成。

gakki：配额放宽是糖，短期留人有效；但真正的护城河是工作流深度，不是上限数字。7 月 13 日之后若回撤，用户迁移成本已经发生，Anthropic 在赌用户不会因为限额的短暂放宽就真的走掉。

## 原文

Anthropic 把 Claude Code 的每周用量上限提了 50%，即刻生效，截止到 7 月 13 日下午 6 点（太平洋时间）。

覆盖 Pro、Max、Team 和按席位计费的 Enterprise 用户，命令行、IDE 插件、桌面端、网页端，所有入口都一样涨。账号已经自动调整，不用做任何操作。

这 50% 是叠加在上周刚宣布的 5 小时窗口翻倍之上的，两个维度的天花板一起被抬。

【注：Claude Code 有两套限额。5 小时滚动窗口管的是短时间内能写多少，写得猛的人一下午就能撞顶；每周总额管的是一周能写多少，防止你某天突然把整周配额刷光。两个一起放宽，等于哪头都松了绑。】

过去几个月 Claude Code 用户最常吐槽的就是 Pro 套餐限额触发太勤，跟 Cursor、Codex 这些竞品比，"用着用着就停"的体感差不少。短期内连开两次额度，留人的意图挺明显。

7 月 13 日之后是否延续，没说。
