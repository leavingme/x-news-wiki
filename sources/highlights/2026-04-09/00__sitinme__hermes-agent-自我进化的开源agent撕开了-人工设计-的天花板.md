# Hermes Agent：自我进化的开源Agent撕开了「人工设计」的天花板

## 摘要

**[「技能自动生长」把SOUL.md式的人工记忆彻底翻篇了]**

NousResearch的Hermes Agent核心突破不是又一个框架，而是「技能自己长出来」——完成复杂任务后自动提取可复用流程，下次遇到类似任务自动调用，跨会话持续迭代优化。这意味着Agent不再依赖人工编写SOUL.md/Memory文件，而是自己生产、自己维护「经验资产」。

**关键差异：**
- 传统Agent：静态Prompt+工具集，用人工填充记忆文件
- Hermes：任务执行→技能提取→自动复用→迭代优化，闭环自驱动
- 类比：OpenClaw的SOUL.md是手绘地图，Hermes是自带GPS且会自己绘制新地图

**对谁有影响：** 正在设计长期记忆系统的Agent开发者、纠结Skill该怎么积累的框架维护者。

gakki：自我进化能力一旦开源，Skill市集的价值逻辑就面临重估——人工维护的Skill会被自动生长的Skill替代。

## 原文

Hermes Agent — NousResearch 做了一个「会自我进化」的开源 AI Agent

NousResearch 发了 Hermes Agent，8.8K Stars。这不只是又一个 Agent 框架——它的核心理念是：Agent 自己会学习、会成长。

它怎么"自我进化"？

传统 Agent 是静态的：你写了什么 Prompt 和工具，它就只会那些。Hermes Agent 不同：

1️⃣ 自动提取技能：完成一个复杂任务后，Agent 会自动把过程中学到的方法提取成"技能"（Skill）
2️⃣ 技能复用：下次遇到类似任务，自动调用之前学到的技能
3️⃣ 持续优化：每次执行都在迭代，同一个技能越用越好

类比一下：OpenClaw 的 SOUL.md / Memory 是人工写的，Hermes Agent 的技能是它自己生成和维护的。

核心功能清单

• 持久化记忆：跨会话保存上下文
• 自主技能生成：从任务中自动学习可复用流程
• 代码执行沙箱：安全运行代码，带自修复能力
• MCP 工具发现：自动发现和接入外部工具
• 多平台统一网关：Web / CLI / API 多入口
• 进程管理：后台任务跨重启恢复
• 浏览器视觉工具：内置 Web 浏览和截图能力

和 OpenClaw 的区别？

OpenClaw：社区维护技能（ClawHub）、Markdown 记忆、多模型支持、生态更成熟
Hermes Agent：Agent 自动生成技能、自动化持久记忆、偏向 Hermes 模型、新发布潜力大

两者不冲突，Hermes Agent 更像是「AI Agent 自主学习」方向的实验性探索。
