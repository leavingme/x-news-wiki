# 小红书 110 万条 Build in Public 背后：中文独立开发者的「反资历」生态正在成型

## 摘要

**[ 3.5 亿活人平台如何用分发机制重塑开发者文化 ]**

• 千粉以下创作者能获得专业反馈——这是小红书独有的「平权分发」机制，GitHub Issue 和 Twitter 做不到。
• 16 岁开发者靠赛事奖金养活团队、00 后 QRBTF 项目从小红书火到 GitHub 7K 星——圈层突破有路径。
• 清华教授到 AI 研究员在这里做 AMA，110 万条内容、3.7 亿阅读，生态已非点状。

gakki：小红书正在成为中文互联网「Build in Public」的最优解——不是因为它刻意设计，而是因为平台的「高审美用户 + 即时反馈 + 无视资历」恰好匹配独立开发者最稀缺的东西。

## 原文

深挖小红书生态：不止黑客松，藏着太多创新惊喜
如果你对小红书的认知还停留在“种草”和一场比赛，可能错过了它正在生长出的生态——它不只是搭台办赛，也在构建独立开发圈层、推动AMA深度互动、提供全链路支持，成为中文互联网“Build in Public”的重要阵地。

小红书对创造者的支持不止于单一赛事。2026黑客松巅峰赛将于4月7日在上海张江科学会堂启幕，200位选手、48小时极限创作、50万奖金池，报名通道3月19日至30日已开放。去年的独立开发大赛，金奖作品QRBTF出自一支00后团队，他们把二维码和AI生图结合，项目在GitHub上获得近7K星。

16岁开发者冼星朗借钱上架产品，靠赛事奖金养活团队；00后Dreamoo梦境社交团队，用5万奖金继续推进创意项目。

“Build in Public”内容在小红书持续增长。截至2025年底，站内相关内容累计超过110万条，相关话题阅读量突破3.7亿。从清华教授到AI研究员，从独立开发者到行业专家，在这里交流互动。平台的平权分发机制让千粉以下的创作者也能获得反馈，评论区常见专业讨论，对开发者而言，这些反馈本身就是产品迭代的参考。

还有一些案例值得关注。赵纯想的AI饮食记录应用「胃之书」，靠小红书用户自发传播，相关邀请码笔记超过10万篇；花叔为女友开发的“小猫补光灯”，在用户反馈中迭代，最终登顶App Store付费榜。从需求发现到产品验证，全链路在这个平台上完成。

小红书通过赛事、AMA互动、用户反馈机制，为开发者提供了从想法到落地的支持路径。这些能力不是刻意宣传的结果，而是由开发者案例和平台数据共同呈现的生态事实。
