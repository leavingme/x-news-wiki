# Step 3.7 Flash 把 Agent 全链路塞进 Flash 档：400 TPS + 一毛钱一次，国产多模态抢了工具编排的活

## 摘要

**StepFun 的 Flash 模型把多模态、搜索、推理三件套内聚到一个端到端流程，速度和成本同时击穿——Agent 编排护城河被原生模型吃掉了第一块。**

- **速度**：400 TPS 输出，号称比 DeepSeek-V4 Flash 快 2 倍以上，且是 Flash 档位而非满血模型。
- **一体化**：截图读图 → 自动生成搜索 query → 视觉提取 + 检索综合 → 结构化竞争分析，决策链不丢给外部工具编排。
- **成本**：实测 Cursor 定价页，不到 1 分钟跑完全流程，单次成本一毛多，Agent 调用的边际成本逼近零摩擦。
- **产业信号**：当 Flash 模型开始内嵌搜索 + 推理，国产厂商在"组装式 Agent 架构"上抢了第一波定义权。

**gakki 锐评**：这是"模型即 Agent 底座"路线的真正拐点——护城河从工作流编排转向原生模型能力。但内嵌搜索的持续性与抗幻觉能力是 3 个月的检验期，StepFun 是否会从黑马变新瓶，需要后续 Agent 长任务的失败率数据说话。

## 原文

发现一个很离谱的模型，400 TPS 的输出速度，比 DeepSeek-V4 Flash 快 2 倍多

还有超强的多模态理解 + 搜索检索 + 推理综合，这居然是一个 Flash 模型能同时做到的

我让它帮我写了一个竞品分析 Agent：
输入一张产品官网截图，自动完成"读图提取 → 搜索市场数据 → 输出竞争分析"的全流程。

然后用 Cursor 的定价页试了一下，不到一分钟跑完，花了一毛多：
它先读懂了整个页面（四档定价 × 各档功能差异 × 推荐标签与社会证明），提取出完整的结构化数据。
然后自己决定搜什么，自动生成搜索 query 去查竞品定价、用户评价、市场对比。
最后综合视觉提取 + 搜索结果，给了我完整的竞争分析：定价对比、竞品定位差异、真实壁垒在哪里、切入建议。

这个模型就是 @StepFun_ai 的 step-3.7-flash，多模态 + 搜索 + 推理一体化内嵌在模型里，如果你在构建 AI Agent、编码工作流或多模态系统，Step 3.7 Flash 很推荐大家尝试一下：
