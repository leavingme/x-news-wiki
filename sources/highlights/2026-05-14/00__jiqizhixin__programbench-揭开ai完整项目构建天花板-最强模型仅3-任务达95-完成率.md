# ProgramBench 揭开AI完整项目构建天花板：最强模型仅3%任务达95%完成率

## 摘要

**【一句话核心判断】
当前AI Agent离"从零构建完整项目"的能力天花板还极远，连头部模型的完整任务解决率都趋近于零。

**要点：**
- ProgramBench 由 Meta FAIR + 斯坦福 + 哈佛联合发布，测试标准极为严格：给文档、用模糊测试验证行为等价性，但不规定实现结构——这是真正考验"理解需求并独立构建"的能力
- 测试的 9 个模型全军覆没，最好成绩是某模型仅在 3% 的任务上达到 95% 完成率，且没有任何模型完整解决过哪怕一个任务
- 结论指向一个被广泛忽视的事实：当前 Agent 的能力边界在"小修小补"而非"从无到有"，吹嘘"AI 写 10 万行代码"的项目往往有大量人工接手

这个结果对整个 AI Coding 叙事是冷水——如果连顶级实验室的联合评估都显示完整项目构建近乎不可能，那实际落地中的应用价值显然被严重高估了。

**gakki：ProgramBench 的出现等于给"AI 颠覆软件开发"的叙事画了一条硬边界。更有意思的是，这条边界不是被行业媒体发现的，而是被学术机构用可复现的基准测试测出来的——说明 Agent 能力的真相迟早会被系统性地量化，而量化才是真正的照妖镜。**

## 原文

Can AI build an entire software project from scratch, not just fix one bug?

Researchers at Meta FAIR, Stanford, and Harvard introduce ProgramBench. 

This benchmark tests if language-model agents can take a program’s documentation and build a full codebase that behaves identically to the original—using automated fuzzing to check behavior without dictating structure.

The result? None of the 9 models tested solved a single task completely. The best model passed 95% of tests on only 3% of tasks. Models prefer monolithic, single-file solutions, far from how humans write code.
