# Bloome 把 Agent 变成「加好友」：扫码即用、群聊可用、还能付费订阅——零门槛分发正在绕过 CLI 护城河

## 摘要

**Agent 分发的最小摩擦单元不是安装包，是二维码。**

- Bloome 让用户把调教好的 Agent 生成二维码，对方扫码即用，无需理解 CLI / Claude Code / 环境配置
- 支持群聊场景：多 Agent 协作 + 「悄悄话」私聊模式，解决了 OpenClaw 多 Agent 抢话的老问题
- 发现页已有人在卖专业 Agent（段永平），Agent 商品化从「能做」进入「能卖」阶段

gakki 锐评：这才是 Agent 民主化的真命题——不是让更多人会写 Agent，而是让不会写的人也能用上别人调教好的 Agent。Bloome 踩中的不是技术问题，是分发问题。

## 原文

我做了很多个 skills，但我妈一个都用不了。

并非 skills 不好用，她根本不知道什么是 Claude Code，什么是 skills。

所以我在想：有没有可能，让 agent 像"加好友"一样简单？

你调教好一个 agent，生成一个二维码，对方扫码就能用。

这周在社群看到有人转了个产品海报，有个叫 Bloome 的内测项目。

打开之后我才明白它在解决什么问题：让不用 Claude Code 的人，也能用上你调教好的 agent。

操作很简单：
1、你在 Bloome 里创建一个 agent
2、将挂载本地的 Claude Code 或 Codex 配置好
3、然后就可以像加好友一样分享给别人

对方不需要懂什么是 CLI，不需要配置环境，直接在聊天界面里 @ 你的 agent 就能用。

甚至可以把 agent 拉进群，一群人一起用。

我之前搞OpenClaw多Agent的时候，就遇到Agent之间相互抢话的问题，他们这块做的很好，而且群里还可以给 agent 发"悄悄话"。

可以被加好友、被拉群、被付费订阅。发现页里已经有人在卖专业 agent 了：段永平、张爱玲、Yiming Zhang……5美金解锁私聊。

你甚至可以买一个 Claude Code Max（200美金），通过 Bloome 共享给10个人用，session 独立。

我整体体验的感受：
这种形式必然是Agent发展趋势，让你的 Agent 更容易传播和分享。
另外，这里可以很方便实现多Agent协作，对我来说太刚需了，我总是害怕一个Agent的回复太片面，如果有各种Agent从不同角度回复，那必然可以大幅提高效率、准确率。

推荐大家快来试一试：https://t.co/kGEzTLz5JI
