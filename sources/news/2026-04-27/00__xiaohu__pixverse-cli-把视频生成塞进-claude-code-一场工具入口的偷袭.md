# PixVerse CLI 把视频生成塞进 Claude Code：一场工具入口的偷袭

## 摘要

**一句话核心判断：视频生成正从「浏览器操作」迁移到「终端调用」，Agent 工作流闭环又补上一块拼图。**

PixVerse 上线的 CLI 工具支持 Sora 2、Veo 3.1、Grok Imagine 等多模型统一调用，安装仅需两秒，可无缝嵌入 Claude Code 和小龙虾（OpenClaw）环境。更关键的是，手机端可通过 Telegram 或飞书远程触发任务——视频生成第一次实现了「人在外面，Agent 在家里跑」的完整闭环。



**gakki 的锐评：** 这不是视频工具的进步，是 Agent 工具链「最后一公里」的打通。但 CLI 化也意味着视频生成从「创作行为」彻底变成「可编程函数」，内容生产的权力结构正在被悄悄改写。

## 原文

简单测试了 PixVerse 出的AI视频生成 CLI 工具

一行命令生成视频，不用开浏览器，直接可以在Claude Code和小龙虾里面利用Agent生成视频

不只是 PixVerse 自己的模型，Sora 2、Veo 3.1、Grok Imagine 模型都支持，都能通过同一个 CLI 调用...

安装就两秒：npm install -g pixverse

就可以输入要求或者提示词让其生成视频，如果绑定了telgram或者飞书可以手机遥控...

也支持图片生视频、AI 配音、唇形同步、音效、超分辨率，基本上网页版能做的，CLI 全能做。
