# Codex 生成 + ChatGPT 润色的组合工作流正在成为 AI 编码的标准中间态

## 摘要

**[AI 工具链正在形成「生成→润色」的双模型分工]**

宝玉分享了一个具体工作流：Codex 生成代码内容，粘贴进 ChatGPT 加一句手绘教育信息图的提示词，产出最终结果。这代表一种新兴的分工模式——专用模型负责代码生成，通用模型负责创意加工。

这和之前 Claude Code 被指「闭源经济学」是一体两面：Claude 的护城河在创意层，Codex 的优势在执行层。

**gakki锐评：** 这个工作流的本质是用 ChatGPT 绕过了 Claude 的多模态限制。问题是——OpenAI 自己的多模态模型也能干这事，为啥还要绕一圈？答案可能是 ChatGPT 的风格化渲染比 GPT-4o 更可控。
