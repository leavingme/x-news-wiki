# AI Agent 调用工具的正确时机：ICML 2026 研究给出「认知必要性」判断框架

## 摘要

**【AI Agent 调用工具的正确时机：ICML 2026 研究给出「认知必要性」判断框架】**

**核心判断：工具调用不是 Agent 默认行为，而是认知缺口时的精准补位。**

**要点：**
- 爱丁堡、CUHK、UIUC 等多校联合提出 Theory of Agent (ToA)，回答了 Agent 何时该调用外部工具这个根本问题：仅当任务无法靠自身内部推理和上下文可靠完成时（epistemically necessary）
- 该框架有效切断了过度思考和过度行动两类失败模式，优于当前主流 Agent 设计
- 通过抑制不必要的委托、强化内部推理质量来提升 Agent 整体表现

**gakki 锐评：** 当行业还在争论「Agent 必须会调用工具」时，这篇论文给出了相反的锚点——内部推理的深度才是 Agent 能力的真正的边界。工具调用率不是能力指标，认知边界清晰度才是。这对当前「堆工具」的 Agent 设计路线是一个温和但有力的修正。

## 原文

When should AI agents actually reach for external tools?

Researchers from Edinburgh, CUHK, UIUC, Northwestern & Princeton present Theory of Agent (ToA) at ICML 2026.

Their answer: only when epistemically necessary — meaning the agent cannot complete the task reliably using its own internal reasoning and context alone.

This framework cuts failure modes like overthinking and overacting, outperforming current agent designs by curbing unnecessary delegation and fostering stronger internal reasoning.

Position: Agent Should Invoke External Tools ONLY When Epistemically Necessary
