# cc-switch升级Codex接入第三方AI：同一引擎下的Harness对决

## 摘要

**【一句话核心判断】**
cc-switch v3.16.0支持Codex接入第三方AI引擎，首次提供了在同一AI基座上横向对比Claude Code和Codex两套Harness工程能力的可能性。

**要点：**
- cc-switch升级到v3.16.0，Codex可接入任意第三方AI
- 测试场景：在相同AI引擎（如Kimi 2.6）下对比Codex vs Claude Code
- 核心命题：两套Harness工程体系谁的体验更优，与底层模型无关

gakki锐评：这是Harness工程学的一次公开PK。当底层模型可以被替换，真正比拼的就是工具设计、对开发者意图的理解、以及上下文管理。Claude Code vs Codex的正面较量，比任何benchmark都有说服力。

## 原文

cc-switch升级到了v3.16.0，Codex也可以随意接第三方AI了。我倒是看看在同样的AI引擎下（比如Kimi 2.6），Codex和Claude Code这两套Harness，到底哪个好。
