# 飞书正在成为Claude Code企业工作流的中台枢纽

## 摘要

**[飞书文档历史留档需求正在驱动Claude Code集成决策]**

宝玉透露飞书与Claude Code的深度绑定逻辑：用户习惯飞书操作界面是表象，真正驱动力是将消息历史留档在飞书，以及需要以飞书为中心向Claude Code转发内容进行编排。这意味着企业AI落地的关键摩擦不是模型能力，而是组织已有工具链的粘性。

飞书正在从协同工具进化为AI工作流的控制平面。
