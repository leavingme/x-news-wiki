# 铁锤人的「编程劝退」：没事别编程——Vibe Coding 的极限反思

## 摘要

**[一句话核心判断]**

Vibe Coding 的「一泻千里」快感之后，铁锤人给出了一个反直觉的终极建议：没事别编程。

**信息增量：**
- 亲历两种范式（手敲代码 vs Vibe Coding）后的对比，验证了 Vibe Coding 的速度幻觉——前期爽，后期维护代价被低估
- 「不要编程」的结论不是反技术，而是对工具错配的清醒认知：非工程师用 AI 生成代码，本质是用不稳定的幻觉替代稳定的无知

**影响：**
- 对独立开发者的警示：AI 降低了编程门槛，但没降低软件工程的专业壁垒
- 加速市场分化：真正需要编程的会更强调工程纪律，不需要编程的会更倾向于购买成品而非自建


**gakki 锐评：**
铁锤人的「劝退」比大多数「AI 编程万岁」的推文更有价值——因为他是真的两种都深度用过。Vibe Coding 的真正问题不是生成代码，而是生成之后谁来维护那个认知赤字？独立开发者的最优解可能不是学编程，而是学会选择不编程。

## 原文

我体验过古法编程，手敲代码，也体验过Vibecoding，一泻千里

我的结论是没事别编程
