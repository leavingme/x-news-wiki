# Claude Code Skills 资源地图：四类入口把官方/平台/创作者/垂直场景全覆盖

## 摘要

**[土豆本豆整理 Claude Code Skill 资源清单，四类分类法揭示 Skills 生态已从 prompt 包装演化为类似 npm 的包生态]**

- 第一类（官方/平台）：Anthropic 官方 12 个 Skills、skillsmp 收录 8w+、skills.market 超市、YouMind 内容创作、Figma 设计系统
- 第二类（社区精选）：claude-code-skill-finder、awesome-claude-skills 等索引型仓库
- 第三类（垂直场景）：设计、测试、文档、PPT 等细分工作流
- 第四类（个人创作者）：宝玉、归藏等 KOL 作品
- 隐性信号：Skills 数量从个位数到 8w+，分发渠道开始分层，类比 npm 的早期形态


gakki 锐评：清单本身会快速过时，但"四类"分类法是有用的认知脚手架。真正的判断是 Skills 生态会不会重蹈 GPTs 覆辙——平台功能搬运一阵后归零，关键看 Anthropic 给不给出生态护城河。

## 原文

我整理了一份 Claude Code Skill 资源清单。

基本涵盖了目前值得关注的 Skill 仓库、Skill 商店和个人创作者作品。

我把这些资源大概分成四类：

一、官方和平台类

1. Anthropic 官方 Skills
包含 skill-creator、Office 三件套等 12 个官方 Skill。
GitHub: https://t.co/bIbHZxVJFQ

2. https://t.co/1Ro2dzD2V8
Skill 超市，有 Trending 榜单，适合快速找热门 Skill。
https://t.co/gMl1g5Alo8

3. skillsmp
收录 8w+ Skill，数量非常多，适合做资源检索。
https://t.co/tzt9lCszDP

4. YouMind Skills  @YouMind_AI
偏学习、写作和创作场景的 AI 技能页，里面有推荐技能包、精选技能和月榜，适合找内容创作、阅读思考、个人简报这类工作流。  
https://t.co/SYB89gKBe5

5. Figma 官方 Skill
偏设计系统和设计工作流。
https://t.co/xl6fnDZP0w

二、开发者最佳实践

1. Superpowers
资深开发者的完整软件开发方法论，基于一系列可配置 Skill 和初始说明。
GitHub: https://t.co/U8uMjpcDVl

2. everything-claude-code
Claude 大赛冠军配置，可以参考它怎么组织 Claude Code 工作流。
GitHub: https://t.co/JdlXE6aeU7

3. gstack
偏开发工作流的 Skill 集。
GitHub: https://t.co/LisN8BWOMP

4. GSD / Get Shit Done
作者在 README 里说得很直白：很多 spec-driven 工具是给 50 人工程团队设计的，而他只是想把东西做出来。
GitHub: https://t.co/uH0FRSHYHv

5. MiniMax Skills
覆盖前端、全栈、Android、iOS、Shader 等方向。
GitHub: https://t.co/xNUMSVuPXY

三、个人创作者出品

1. 李继刚 Skills @lijigang
覆盖卡片、论文、学习、写作等场景。
GitHub: https://t.co/IXwMl4aQWn

2. 宝玉 Skills  @dotey
包含翻译、配图、发布等工作流。
GitHub: https://t.co/Z1OauorRYP

四、两个我觉得值得单独看的 Skill

1. frontend-design
强制Claude跳出AI式安全美学，先选美学方向（极简、复古未来、奢华），再选独特字体组合和大胆配色，拒绝Arial/Inter这种万金油。
GitHub: https://t.co/bIbHZxVJFQ

2. claude-mem
不用每次开新会话都重新解释你的项目。对长期项目很有用，尤其是 Claude Code 这种需要持续理解项目结构、约定、上下文的场景。
GitHub: https://t.co/XvmG6LknFk
