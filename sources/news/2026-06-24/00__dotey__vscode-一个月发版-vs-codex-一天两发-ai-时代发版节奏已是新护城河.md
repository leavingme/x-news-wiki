# VSCode 一个月发版 vs Codex 一天两发：AI 时代发版节奏已是新护城河

## 摘要

**宝玉指出 VSCode 一个月发一个版本，Codex 一天发 2-3 个版本，且 VSCode 本身也开始卡顿。**

- 传统 IDE 的季度/月度节奏在 Agent 时代已显疲态，迭代速度本身成为产品竞争力
- 暗含一个判断：AI Coding 工具的竞争已从「功能完备」转向「高频试错 + 用户共创」
- VSCode 卡顿现象侧面印证 Electron 架构在 AI 增强场景下的承载瓶颈


gakki 锐评：发版速度本质是「组织学习速度」的外化，Codex 的高迭代率建立在 OpenAI 内部闭环之上——这恰恰是 Anthropic 用 Claude Code 也在抢的阵地，慢公司会越来越慢。
