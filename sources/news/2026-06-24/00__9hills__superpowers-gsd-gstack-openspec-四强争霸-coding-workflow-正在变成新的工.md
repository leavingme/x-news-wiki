# superpowers / gsd / gstack / openspec 四强争霸：Coding Workflow 正在变成新的工程范式战场

## 摘要

**Coding Workflow 框架迎来项目级爆发，独立开发者开始自组工作流操作系统**

- 四个明星项目分别覆盖不同的 Workflow 取向：superpowers（能力包）、gsd（任务分解）、gstack（YC 背书的工作流 OS）、openspec（规约先行）
- 九原客的态度信号：从单一项目转向「汲取 Skill 自组 Research→Design→Plan→Develop→Test→Commit」流水线
- 意味着 Harness Engineering 的工程范式正在从「框架大一统」走向「Skill 拼装」


gakki 锐评：Workflow 项目井喷的本质是 Claude Code 把 Agent 能力标准化后，差异化竞争被推到「怎么用」这一层——未来胜出的不会是框架本身，而是谁的 Skill 库被更多开发者抄作业。
