# dbskill 亲述：免费开源 Skills 的隐性收益是直接卖课的百倍

## 摘要

**[好产品 ≠ 好商品：免费开源的 Skills 比付费卖更有商业价值]**

- dbskill 开发者自述受 Garry Tan 启发，公开全部 Skills 资源，直接/间接收益是直接卖课的 10-100 倍
- 卖 Skills 的三重死结：付费后用户还要装 Agent → 售后灾难；toB 定制改到开发者自己不想维护；无论怎么卖都卷不过一键爆款智能体
- 核心洞察：「好产品、好商品、赚钱快的商品是三类不同的东西」，附带效应（0 佣金带货 100+ 台 MacBook）才是真正的现金流


gakki 锐评：这是 Skill 时代的第一批商业教训——「卖铲子」思路在 Agent 生态里跑不通，因为「铲子」本身可被零成本复制，溢价只能从信任/流量/带货等周边业务里挤。

## 原文

之前有人问为什么要开源 skill，而不是拿去卖

我说因为我是收到 GarryTan 启发，他的提示词没收我钱，那我也不对外收钱

而且本来 skills 就是用推文整理出来的，推文也是免费公开的。不怕费 token 的话，大家都可以开个 AgentTeam 把 dbskill 整理出来

不过最近我可以确信了，dbskill 带来的直接/间接收益，比直接卖 skills 多赚了十倍百倍

另外多说两句，直接卖 skills 其实也不是个好生意

1、用户认知成本太高，大家为了白嫖 skill 可以去安装 Agent，甚至去买一台 Mac（我已经 0 佣金带货 100 台以上 MacBook 了），但是你要是让买家花了钱，还得装上 Agent 才能用，实际会产生售后问题

2、dbskill 我有自己的思路，反正是不收费，我往哪开发别人也管不着，但是如果 toB 去卖，就得按他们的思路魔改，最后会改的我自己都不想维护，会变成短命项目

3、最终你折腾一圈，会发现，无论如何你卖不过一键爆款智能体

好产品、好商品、赚钱快的商品，这是三类不同的东西

而我们只是偶尔做一下第三类，多数情况下其实我们做的是前两类
