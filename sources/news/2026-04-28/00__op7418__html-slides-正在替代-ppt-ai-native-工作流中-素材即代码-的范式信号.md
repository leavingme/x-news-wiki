# HTML Slides 正在替代 PPT：AI Native 工作流中「素材即代码」的范式信号

## 摘要

**[一句话核心判断]**

在 AI Native 工作流里，PPT 正在被「HTML 网页 + GPT-Image-2 生成素材」的组合加速替代，这不是格式之争，而是创作管道的根本重构。

**信息增量：**
- PPT 的本质是「幻灯片」，而 AI 时代的工作流核心是「可编程、可迭代、可自动生成的素材管道」
- GPT-Image-2 生成素材图 → 嵌入 HTML 页面 → 直接分享，这一链路比导出 PPT 再分发少了两个摩擦节点
- 素材图可单独更新，无需重建整个 PPT 文档版本

**影响分析：**
对内容创作者和分享者影响最大：传统 PPT 需要完整版本控制，HTML Slides 天然支持增量更新和 Git 式管理。但设计复杂度和兼容性仍是企业场景的阻力。

gakki锐评：AI Native 圈默认 HTML Slides，本质是「代码即文档」哲学向内容领域的渗透。PPT 不会死，但 PPT 作为信息密度最高的交付格式时代正在结束——不是被干掉，是被解构进了更快的管道里。

## 原文

说一下 PPT 本身还是 HTML 网页，只是里面的素材图会用 GPT-Image-2 0 去生成
