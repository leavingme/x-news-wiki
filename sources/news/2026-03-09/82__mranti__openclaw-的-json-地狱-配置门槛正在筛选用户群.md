# OpenClaw 的「JSON 地狱」：配置门槛正在筛选用户群

## 摘要

**[Agent 工具易用性瓶颈的真实反馈]**
- Michael Anti 吐槽：阿里云百炼等模型必须手动改 4 个 JSON 文件，括号错一个就死
- 核心痛点：缺乏「怎么改都不会错」的 JSON 编辑器，非代码用户上手困难
- 与 Leobai 观察形成镜像：配置痛苦 → 部署服务有市场

**gakki 锐评**：这是典型的「开发者体验债务」。OpenClaw 的 Skill 系统本意是降低门槛，但配置层的 JSON 地狱又把门槛抬高。GUI 配置界面不是「锦上添花」，而是决定 Agent 能否跨越鸿沟的关键基础设施。

## 原文

我最近也配置了不少（次）openclaw，我说下最痛苦的是什么，其实是很多配置必须通过手动改4个json文件解决（比如阿里云百炼系列模型只有手动改一条路）。而且json文件那么多左右括号，你搞错一个就死。咋就没很好用的json编辑器？那种怎么改都不会错的？一句话：小龙虾真不太适合不会代码的人啊。
