# Claude Code正在瓦解硅谷工程师的「协作契约」

## 摘要

**[AI coding工具正在从根本上改变软件工程的分工逻辑与代码审查机制]**

• 湾区AI创业公司面试门槛飙升：斯坦福/藤校本科+2-3段实习成junior标配，入职即被要求产出senior级别代码
• 工程师模块独立性被迫增强：当每个人都用Claude Code写代码，代码溯源困难导致「谁写的」不再重要，「谁负责」成为唯一问责维度
• 隐性成本转移：团队省下的coding时间，正在转化为更高的沟通成本和架构维护负担

**gakki锐评：** 这不是「AI提升效率」的美好叙事，而是组织在工具冲击下的应激重组。当代码生成变得不可追溯，工程师的价值正从「写代码」被迫转向「为AI的产出背锅」——而大多数人还没准备好承担这个代价。

## 原文

今天跟在湾区做engineer的一位大学同学聊天，他在一个100+员工的AI infra创业公司工作。

他给我说，最近来面试他们公司的junior engineer基本都是斯坦福、藤校之类的本科生，而且需要2～3段实习经历，一来能够直接干原本senior的活。

在湾区大厂工作相对还是比较舒服的，但在创业公司，现在工作量很大。如果不快速学习出活儿的话，很容易被优化掉🥲

另外因为大家都在用Claude Code写码，且已经挺放心AI的了，以至于每个人的码怎么写出来的不好相互解释，所以反而每个engineer负责的模块都变得更加独立了😂
