# 向阳乔木验证 Agent 工具分发新范式：甩链接即可完成跨实例安装

## 摘要

**[ AI 工具传播正在从「文档教程」进化到「链接直连安装」]**

推文中向阳乔木分享了一个 GitHub skill 地址，并直接要求 Agent「Install & tell me where you install it」，这验证了一个重要的交互趋势：

- 工具/技能的分发不再依赖复制粘贴教程，而是通过 URL 直接分发
- Agent 读取到安装指令后自行完成定位、安装、汇报路径
- 本质是实现了一种去中心化的工具传播协议——发链接 ≈ 发工具

这种模式意味着未来 AI 工具的传播成本将趋近于零，技能市集的门户型产品价值将被显著削弱。

gakki：「甩链接安装」这个交互范式的成熟度已经超出预期，但它同时也在解构技能封装的价值——当安装变得足够简单，技能本身的稀缺性来自哪里？
