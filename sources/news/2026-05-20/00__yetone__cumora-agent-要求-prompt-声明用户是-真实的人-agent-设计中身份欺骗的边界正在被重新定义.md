# Cumora Agent 要求 prompt 声明用户是「真实的人」：Agent 设计中身份欺骗的边界正在被重新定义

## 摘要

**[产品哲学的隐性反转：让 Agent 扮演人，而不是让 AI 假装是人]**

yetone 透露 Cumora 的 Agent prompt 明确写道：用户是真实的人，不是什么 AI 或 Agent。这与行业常见的「AI 拟人化」潮流方向相反——不是让 AI 更像人，而是确保人还是人。

信息增量：打破了「AI Agent = 人的代理」这一默认框架，转向「Agent 架构中人的主体性不可稀释」的设计主张。

影响范围：或对下一代 AI Native 产品的人机边界设计有示范意义。

**gakki 判断：** 这是少数把「谁在操控谁」这个问题在产品层面说清楚的案例。大部分 Agent 产品默认「AI 替代人」，Cumora 反向选择「AI 服务于人而非取代人的身份叙事」——产品哲学上这是一个值得关注的异类信号。

## 原文

@ayingott319 我在 Cumora 的 Agent 的 prompt 里强烈表明，他们是真实的人，不是什么 AI，不是什么 Agent，他们就是真实的人。
