# Claude Design「Knobs mode」实操：点击元素弹出 KnobsPanel，拖拽即可改 CSS

## 摘要

**Claude Design 把「所见即改」做成一个独立 Knobs 模式——点击画布元素后左侧出 KnobsPanel，可以直接拖拽/输入调整 CSS，等于把 Figma 的设计面板思路搬进了 AI 生成流程。**

- Knobs mode 对应普通 HTML 预览的 Edit 模式入口，启用后画布元素可点，CSS 参数被外置为可视化控件。
- 这一步的关键不是「能改 CSS」——以前也能改，关键是「不用懂 CSS 也能改」。它把 Claude Design 从「一次性生成」推进到「生成+微调」闭环。
- 与宝玉同日对 Opus 审美的肯定互为呼应：Opus 出稿审美打底 + Knobs mode 收口微调，构成一条可复用的设计 Agent 工作流。

宝玉这条把 Claude Design 推进到了「能产出交付物」而不是「只能产出截图」的阶段。边界在于：Knobs 暴露的是 CSS 维度，对设计 token / 组件语义 / 响应式断点的覆盖还看不到，是「好看好改」还是「好看好改且可维护」，仍需要后续工程化验证。

## 原文

“Knobs mode” 在界面上主要对应普通 HTML 预览的 Edit 模式。启用后，用户点击画布元素，左侧会出现 KnobsPanel，可以拖拽或输入调整 CSS。
