# Claude Desktop Side chat 袖珍到无法浏览：默认大屏假设是桌面 Agent 的隐性傲慢

## 摘要

**[Anthropic 桌面端 UI 明显没考虑普通屏] 宝玉吐槽 Claude Desktop 的 Side chat 面板过小、内容无法直接复制。**

- 宝玉原话：'设计出来的 Side chat 才这么袖珍，根本没法好好浏览，甚至内容都无法直接复制出来'。
- 反映 Claude Desktop 团队对使用场景的默认假设（开发者一定有超大外接屏）与实际用户（笔记本/普通分辨率）严重错位。
- 复制内容需要靠 Side chat 是高频操作，禁用选择/复制等于把对话关进玻璃箱。


gakki 锐评：桌面 Agent 现阶段最大的敌人不是模型能力，是产品经理的'自家工作站幻觉'——Chat UI 范式固化的另一面是 Chat UI 体验的全面降级。
