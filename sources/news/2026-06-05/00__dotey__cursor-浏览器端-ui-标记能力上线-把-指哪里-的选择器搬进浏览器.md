# Cursor 浏览器端 UI 标记能力上线：把"指哪里"的选择器搬进浏览器

## 摘要

**Cursor 把"哪里要改"的选择器从代码编辑器搬进了浏览器，AI Coding 进入"圈选即改"阶段**

要点：
- 浏览器端直接对 UI 元素做视觉标注，免去"截图→描述→定位代码"的三段式损耗
- 与 AI 编码能力联动，圈选元素即可触发修改意图传递
- 对前端协作流程：设计师-开发者的标注语言首次被原生 AI 工具吸收
- 行业信号：v0/Lovable 的"重新生成"路线 vs Cursor 的"在原代码上精准改"路线开始分化

gakki 锐评：这是 Cursor 对"AI 重写一切"叙事的克制反驳——不取代现有代码库，而是把标注层嫁接上去。但浏览器圈选 ≠ DOM 语义理解，复杂组件（Ant Design / shadcn 等）的指代精度仍是工程化关键。

## 原文

Cursor 也支持在浏览器端标记 UI 了，这样修改 UI 起来就方便多了
