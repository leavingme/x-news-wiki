# Chichan 一句话戳穿 AI Coding 工具的 npm 供应链死结：装包即授权

## 摘要

**AI 编程工具让恶意 npm 包的杀伤半径从本地 dev 环境扩大到完整应用栈。**

- 核心洞察：根问题不是环境变量泄露，而是你根本无法避免装入恶意包——这本身就是一个零信任盲区
- 在 AI Agent 时代，Agent 自动 install、auto-run 代码，攻击面被进一步放大
- 解决思路正在从「检测单个包」转向「运行时隔离 / 沙箱」或「白名单依赖」


gakki 的锐评：这是 Vibe Coding 时代的隐性税——AI 写代码速度越快，供应链信任的隐性负债堆得越高，本地 + 沙箱部署会从「可选」变成「必选」。

## 原文

@mattpocockuk I mean, at this point, you're just running a malicious package that has access to your entire app. So, the root problem here isn’t the exposure of environment variables, but rather how you avoid installing such packages.
