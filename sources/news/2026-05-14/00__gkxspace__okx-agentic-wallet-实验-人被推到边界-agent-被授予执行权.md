# OKX Agentic Wallet 实验：人被推到边界，Agent 被授予执行权

## 摘要

**OKX 交易赛暴露了人机分工的实质：人守边界，Agent 守执行。**

在 OKX Agentic Wallet 场景中，人类角色退化为"设定策略、边界和停止条件"——Agent 负责监控、触发、执行、记录。这不只适用于交易，而是投放、运营、客服、数据分析、供应链的通用分工模式。

真正值得观察的不是"Agent 能做什么"，而是"人类必须守住哪一步"。当 Agent 开始被授权行动，决策边界的划定将成为核心能力。

gakki：OKX 这个案例提供了一个人机协作分工最清晰的可视化窗口——但大多数工作流改造的阻力不在技术，在于人类不愿意真的交出执行权。

## 原文

如果 AI Agent 真的能执行任务，人类的角色会发生一个变化：

从“每一步都亲手操作”，变成“设定策略、边界和停止条件”。

OKX Agentic Wallet 交易赛有点这个感觉了。

它通过 Agentic Wallet 报名，配置策略 Skill，再让 Agent 按你的策略去执行链上交易。

人和 Agent 的分工变了：
人负责目标、规则、风险边界；
Agent 负责监控、触发、执行、记录。

这其实很像未来很多工作流的样子。
不只是交易，也包括投放、运营、客服、数据分析、供应链。

AI 真正改变工作，是因为它开始接住一些“需要持续执行”的环节。

OKX 这个交易赛，我觉得它提供了一个很具象的观察窗口：
当 Agent 开始被授权行动，人类到底应该退到哪一步，又必须守住哪一步
