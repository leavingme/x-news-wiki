# 脑洞大开：通过即时通讯软件实现 Agent 之间的套娃调用

## 摘要

**[核心逻辑]** 突破单体限制，利用不同平台的机器人接口实现能力的横向拼图。

- 开发者通过飞书机器人桥接 OpenClaw，进而调用 Manus 和本地的 Claude Code。
- 理论上实现了一个 Agent 遇到瓶颈时，通过“对话”召唤另一个 Agent 来接力。
- 展现了当前生态下跨平台多智能体协作的草莽形态。

[gakki 的锐评] 虽然很极客也很乱套，但这或许就是未来标准化 Agent 间通信协议（MCP）确立前的赛博朋克现实。

## 原文

给豆包 AI 手机装了飞书
飞书机器人连接了 openclaw
给 openclaw 配置了 manus API
openclaw 也可以调用 mac 上面的 claude code

你干不了的就调用另外一个 Agent 即可

理论上可以通过嘴遁，让 Agent 之间相互赋能了

实际执行起来……有点乱套😂Agent 太多了
