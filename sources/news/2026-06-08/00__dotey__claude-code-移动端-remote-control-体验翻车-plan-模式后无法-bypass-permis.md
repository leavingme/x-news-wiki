# Claude Code 移动端 Remote Control 体验翻车：Plan 模式后无法 Bypass Permission，频繁确认击穿移动场景

## 摘要

**[一句话核心判断]** Claude Code 的桌面端哲学（plan → confirm → execute）直接搬到移动端 Remote Control 上，是一次典型的「场景错配」暴露。

要点：
- 移动端操作的核心痛点：人在路上，无法承担高密度确认交互
- Plan 之后必须逐条 bypass permission，把 Agent 自主性压回 0，本质上把移动端降级为「带屏幕的 CLI」
- 这暴露出 Anthropic 在多端一致性 vs 场景适配之间还没有真正分层，移动场景需要「信任窗口」或「场景化策略集」


移动端 Agent 的胜负手不是「能不能跑起来」，而是「敢不敢替你做决定」。Claude Code 这次直接搬运桌面权限模型，等于在移动场景下自废武功。
