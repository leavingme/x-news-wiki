# 宝玉：Chat 时代求一次通过，Agent 时代靠错误反馈循环——可调试性正在重写技术栈选型

## 摘要

Agent 时代，「能否被快速修正」比「能否一次写对」更重要。

- Chat 时代无运行环境，模型必须一次通过；Agent 时代有明确错误反馈，迭代修复成本极低
- React/JS 有清晰报错 → 定位快 → Agent 可自主闭环修复
- HTML 内联样式错误无明确信号，是 Agent 调试的「沉默黑洞」

宝玉把工程哲学压成一个产品判断：可调试性不是开发体验问题，是 Agent 友好度的隐性指标。Claude Design 默认生成 React 组件而非 HTML，正是这套逻辑的产品化结果——技术栈选择正被 Agent 生态反向定义，HTML 这类「无显式反馈」的前端范式将逐步被边缘化。

## 原文

在 Chat 时代，追求的是一次通过，因为没有运行环境，但是在 Agent 时代，只要有明确的错误反馈，修正起来很快。

React 代码，js出错了很好调试修复，css错误只要能定位到具体组件也很好修；一个大的 HTML 里面的一些样式细节出错了，不会有啥明显的错误，但是 debug 起来可真的会要命。
