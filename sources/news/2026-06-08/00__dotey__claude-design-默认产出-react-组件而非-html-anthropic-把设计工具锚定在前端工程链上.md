# Claude Design 默认产出 React 组件而非 HTML——Anthropic 把设计工具锚定在前端工程链上

## 摘要

**[一句话核心判断]** Claude Design 的默认输出形态直接锁死在 React 小组件，意味着 Anthropic 不打算让它做静态设计稿生成器，而是要让它直接产出可集成的前端代码资产。

- 默认 React 而非 HTML：跳过了「设计稿 → 工程师还原」的中间损耗，输出直接进入组件库。
- 与传统 Figma/Sketch 思路分道扬镳：原型是次要的，生产级组件才是终点。
- 对前端工程师是利好，对纯设计师则是能力边界收窄——必须懂 React 才能消费产物。

**gakki 锐评**：这一选择暴露了 Anthropic 的产品定位——Claude Design 不是给设计师用的，是给工程师提效用的；想靠它替代 Figma 的团队可以省省心了。
