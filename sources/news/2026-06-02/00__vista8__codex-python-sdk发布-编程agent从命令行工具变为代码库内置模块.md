# Codex Python SDK发布：编程Agent从命令行工具变为代码库内置模块

## 摘要

**一句话核心判断：** OpenAI发布Codex Python SDK（pip install openai-codex），将编程Agent能力直接嵌入第三方代码，改变了Agent工具的存在形态。

**要点：**
- 可复用Codex登录态，降低接入门槛
- 相当于在自有代码中内置顶级编程+生图Agent
- SDK形态意味着编程Agent从独立工具变为可编程模块，这是生态入侵的关键一步

**gakki锐评：** SDK vs CLI是两种生态策略——CLI吸引开发者，SDK收割工程团队。当Codex能力可以被pip install，谁还需要自己造轮子？

## 原文

这个有点厉害，Codex 出 Python SDK了。

安装指令：pip install openai-codex

整合到自己的代码中，相当于直接内置了顶级编程和生图Agent？

最关键的是，可以复用 Codex 登录态。
