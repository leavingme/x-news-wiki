# Grok Build的headless推特搜索被严重低估：给所有Agent加搜索外挂

## 摘要

Grok CLI的headless模式一行命令直接搜推特，完全合规且无API限制，可封装成Skill供ClaudeCode/Codex调用。测试者还发现它自写的提示词过不了Grok审核——说明Grok的内容过滤比想象中严。作者用它尝试生图，结论是'太权威了'。这不是赞美。

## 原文

Grok Build 有个被严重低估的能力：headless 模式，一行命令直接在终端搜推特，而且这是完全合规的用法

这意味着什么你的 ClaudeCode/Codex 原本没有推特搜索能力，接上 Grok CLI 就有了，相当于给所有 Agent 加了个搜索外挂

当然，你也可以将这整个封装成一个 skill 方便 AI 调用

也能这样调生图生视频模型，我尝试了一下，他自己写的提示词图片过不了审核，看来 Grok 还是太权威了......
