# 模型推理能力的隐性分层：Extra High 模式才是隐藏的生产力开关

## 摘要

**[付费墙背后的能力跃迁：从"能用"到"好用"的阈值效应]**
• 同一问题在 Codex 与 Claude Code 多轮未能解决，切换至 ChatGPT Pro 的 Extra High Reasoning 后突破
• 揭示当前模型能力的分层现实：基础模式解决 80% 简单问题，高难度问题需要"深度思考"模式的跨上下文推理
• 开发者存在 Token 节俭幻觉：为节省成本使用低推理强度，反而浪费更多交互轮次
• 提示模型选择策略：当任务涉及复杂架构设计或跨文件依赖时，应优先调用高推理层级

gakki 的锐评：省钱是开发者最大的认知陷阱。为省几美分的 Token 却浪费半小时调试，这是典型的穷人思维。

## 原文

用 Codex 解决一个问题，好几轮都解决不了，换 Claude Code，还是解决不了。突然意识到，我已经是 ChatGPT Pro 用户了，就别省 Token 了，于是设置 Reasoning Level 程度为 Extra High，解决了…
