# SuperPowers 106k Star：多 Agent 并行的工程化纪律正在成型

## 摘要

**[从「AI能写代码」到「AI能按标准工程流程写代码」的跨越]**

• 需求阶段：疯狂提问 → 标准设计文档 → 用户确认，杜绝需求漂移
• 隔离机制：新分支 + 独立工作区，主代码库保护成为硬性约束
• 原子化执行：任务拆分到最小子任务，子Agent执行并强制测试，最小代码也需通过
• 双重审查：设计规格符合性 + 代码质量，任一不通过即阻塞

**gakki锐评**：当「多Agent并行」从炫技变成有纪律的工程流程，Vibe Coding的生产力拐点才真正到来——规则不是限制，是规模化的前提。

## 原文

没发现SuperPowers之前我简直是在乱用AI写代码！

每次写出来的虽然能跑但是架构非常乱
项目一大就全盘崩溃、维护也是地狱难度

SuperPowers制定了一套规则
让AI可以像资深团队一样有一套标准的开发流程。
目前已在Github斩获106kstars

SP的核心流程
1.先疯狂提问，把你的需求完全理解，你同意后并生成标准设计文档

2.开新分支+隔离工作区，确保不会搞乱主代码

3.把任务拆分成最小子任务

4.交给子Agent执行并强制测试，最小代码也要通过测试

5.对完成的任务进行双重审查，不仅要符合设计规格，更要检查代码质量，有一项不过关就不放行

多agent并行操作，给ai说完要求之后，就可以解放双手，喝着咖啡监督agent工作。
