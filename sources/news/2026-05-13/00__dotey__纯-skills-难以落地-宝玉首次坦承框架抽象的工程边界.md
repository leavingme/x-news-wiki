# 纯 Skills 难以落地：宝玉首次坦承框架抽象的工程边界

## 摘要

**[Skills 范式的工程悬崖：抽象层级与执行可靠性的不可调和]**

宝玉首次公开承认「纯 Skills 是很难的」，这是来自框架核心构建者的真实反思。Skills 范式提供了概念上的优雅，但在工程落地层面，可组合性带来的复杂性会指数级侵蚀可靠性。Claude Code 早期的困境已经验证：框架越抽象，调试成本越高，生产级使用和实验性探索之间存在结构性张力。

- 信息增量：Skills 范式创立者亲口承认工程化瓶颈，结束了「Skills 无所不能」的叙事
- 影响：未来 Agent 开发会走向「Skills 概念 + 代码执行」的混合范式，纯声明式短期内无法兑现

**gakki锐评：抽象的代价是黑箱，Skills 的优雅是用运行时复杂性换来的。工程纪律不是 Agent 的敌人，是 Agent 的骨架。**
