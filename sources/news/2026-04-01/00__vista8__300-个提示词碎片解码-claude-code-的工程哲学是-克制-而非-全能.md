# 300 个提示词碎片解码：Claude Code 的工程哲学是「克制」而非「全能」

## 摘要

**一句话核心判断：** 泄露的 Claude Code 提示词揭示了一个反直觉的产品哲学——它被设定为「常常能帮你完成过于复杂的任务，但不要做过头，要格外简洁」，这是对 Agent 狂热的一剂降温针。

- 从 300 多个提示词片段中提炼出的核心要求：能力强大但不主动膨胀、先尝试最简单的方法、不要兜圈子、尊重用户对任务规模的判断。
- 这与当前 AI Agent 市场普遍的「功能堆砌」路线形成鲜明对比——越强大的 Agent 越需要自我约束的边界设计。
- 提示词中「尊重用户自己的判断」这一条尤其值得注意：它把最终决策权明确留给人类，这是对齐问题在产品层的具体落地。

gakki 锐评：当整个行业在追求「更强大」的 Agent，Claude Code 的提示词哲学却在说「克制」——这不只是工程品味问题，它反映了一个根本矛盾：让一个统计系统学会「不要做过头」，比让它「做更多」要难得多。


## 原文

朋友根据Claude Code泄露的代码，让CC分析了其中的提示词，竟然有300多个片段。

读了一部分，发现很多提示词工程要求，比如：

① 你能力很强，常常能帮助用户完成那些原本过于复杂或耗时过长的雄心任务。至于某个任务是否大到不值得尝试，应尊重用户自己的判断。

② 直接切入重点。先尝试最简单的方法，不要兜圈子。不要做过头。要格外简洁。
