# 宝玉定性 AI Slop：开发者反抗的是「AI 写完本人都不看」的 PR 黑洞

## 摘要

**[一句话核心判断]** 宝玉澄清：开发者针对截图类问题抵制的是「AI 写完不看就提 PR」的工程态度，而不是 AI 辅助本身。

- 这是一场工程纪律的边界划定：AI 工具是放大器，使用者的「看一眼」动作不能被取消。
- 反驳「针对非专业开发者」的误读：宝玉明确划线，反对的是「AI 写完不审 PR」，不是反对 AI 辅助。
- 与此前 Vibe Coding 死循环、工程债务等议题一脉相承，再次落到「判断力稀缺」这个核心矛盾。

**gakki 锐评**：这本质上是「工具中性论」的反例——AI Coding 工具的滥用边界不是工具能力，而是工程师的最小审阅纪律；不能审的人用 Claude Code 等于在给项目埋雷。
