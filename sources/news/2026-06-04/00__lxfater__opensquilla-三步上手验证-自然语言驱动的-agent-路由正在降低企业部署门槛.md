# OpenSquilla 三步上手验证：自然语言驱动的 Agent 路由正在降低企业部署门槛

## 摘要

**[自然语言触发 skill + 智能路由派发，Harness 层正在把 Agent 部署变成「说人话」的事]**

上手路径极简：1）一行 uv 命令安装；2）onboard 向导配置 API key 和模型档位；3）大白话说目标，MetaSkill 自动挑 skill 拼工作流。默认省 token 路由按难度自动判别派发，无需人工干预。

**信息增量：**
- 验证了「Harness as a Service」的产品路径：面向非工程师用户的 on boarding 体验正在成熟
- 与 LangChain 的复杂初始化相比，uv tool install 的一行命令代表包管理哲学的范式转移
- 路由表可配置（每档走哪个模型），企业有控制权，不是黑盒


**gakki 判断：** 当安装比文档还简单的时候，Agent 工具的采用曲线才真正从开发者卷到产品经理。Harness 层的易用性战争，才刚开始。

## 原文

其实上手不复杂，三步

1. 安装：一行 uv 命令
uv tool install "opensquilla[recommended]"
没装 uv 的，先 curl 一行把 uv 装上；Windows 还有免 Python 的便携版

2. 配置：跑一次 onboard 向导，填个 API key，选好每档走哪个模型，就上面那张路由表

3. 使用：剩下的不用操心
省 token 路由默认自动判难度派发
MetaSkill 也默认开着，用大白话说目标，它后台自己挑 skill、拼工作流

下面给出地址，你们可以试一试
