# 宝玉定调 Claude Design：DS 才是入场券，UI 自动化让位 UX 设计

## 摘要

**Claude Design 的真正价值不是 UI 像素生成，而是 UX 层面的协作——前提是你已有 Design System。**

宝玉厘清 Claude Design 的适用边界：
- 关键前提：用户必须先有 Design System（DS），UI 规范已在 DS 层固化，Claude Design 才能在 DS 框架内做变体
- 价值位移：没 DS 的团队用 Claude Design 容易陷入"逐页生成 UI"的反模式，DS 团队则能把 Claude Design 用在 UX 流程、交互逻辑、信息架构上
- 方向校准：这是对"Claude Design 颠覆 Figma"叙事的降温——它是 DS 团队的放大器，不是设计系统缺失者的救世主


gakki 锐评：宝玉这条点破了 AI 设计工具的最大幻觉——它不会让"没设计系统的团队"突然拥有设计系统，反而会强化马太效应。真正值得关注的反而是他没说的：DS 不在 Figma/MCP 生态里的团队，Claude Design 几乎用不起来。

## 原文

@saurabh_visator Claude Design 有一个很重要的前提：你有自己的 DS（Design System）。当你有了 Design System，那么你不需要太多的 UI 设计，因为 UI 设计规范在 DS 已经定义好了，更多的是 UX 的设计。这才是正确的方向。
