# Cursor 20亿 ARR 背后：个人开发者正在「叛逃」到 Claude Code

## 摘要

**[AI Coding 市场出现结构性分化]**
• Cursor 企业客户占比升至 60%，但个人开发者持续流失给 Claude Code/OpenClaw
• 开发者工作流演进：从 Tab 补全 → Cursor Agent → Claude Code → OpenClaw 编排，代码本身正沦为「中间产物」
• 国内大厂今年目标集体押注 AI Coding，赛道从尝鲜进入必争阶段

个人开发者永远走在最前面，他们的迁移路径就是市场下一步的风向标。

## 原文

Cursor ARR 突破 $20 亿，3 个月翻倍，ARR 的计算方式我们另说😂

有意思的是个人开发者在流失给 Claude Code，但企业客户占比反而升到 60%。

很明显 AI Coding 工具的战场正在分化，我最近聊了几家国内大厂，今年目标不约而同都是 AI Coding。这个赛道已经从尝鲜变成必争之地。

回过头来说 cursor 的数据，有个很重要的因素是个人开发者永远走在最前面。

回顾我自己的路径：
- 23～24：Tab 补全，觉得已经很神奇了
- 25：Cursor Agent 模式，开始让 AI 写整个函数
- 25 年底至今：Claude Code，除了代码结构设计，已经不想再看代码了

特别是openclaw 出来后，我的工作流是：描述需求 → 跟小龙虾讨论清楚 -> 他把工作派给 claude code / codex / gemini 去实现 → 自动化单元测试 ai review -> 看 diff 确认 → 提交。代码本身变成了中间产物。

Claude Code 走在最前面不是偶然，除了走得快（比如快速跟进了可以远程指挥）外，它的 Agent 能力、上下文理解，我个人体验下来感觉比 Cursor 领先至少半年。

至于 Cursor 企业占比提升到 60%？有几个可能，
1. 个人开发者跑了，这仆人追求最强模型，用脚投票；
2. 部分企业本身行动迟缓，可能刚走到 ide 这一步，动作永远慢一拍。分母变了，占比自然上去了。

这也解释了为什么 Cursor 估值还能到 $293 亿，企业市场的钱更好赚，LTV 更高，流失率更低。

选这条路，商业上没毛病。

但技术前沿技术还在 Cursor 吗，我看不是了，群狼环伺
