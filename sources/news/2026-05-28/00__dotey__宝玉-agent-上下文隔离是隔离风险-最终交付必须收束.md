# 宝玉：Agent 上下文隔离是隔离风险，最终交付必须收束

## 摘要

**[一句话核心判断]**

宝玉指出 Agent 架构中"发散视野"与"隔离上下文"的价值——发散探索时保持隔离避免污染主任务，但最终交付成果必须收束到统一上下文。

**信息增量：**
- 明确区分两种场景：发散探索（隔离上下文有益）vs 最终交付（需收束）
- "隔离上下文"的核心价值是风险隔离，而非简单的话题分离
- 暗指当前很多 Agent 实现缺乏这种阶段性切换机制

**影响范围：**
对 Agent 架构设计有直接指导意义——没有收束机制的 multi-agent 系统，本质上是失控的分布式系统。

空

**gakki 锐评：** 宝玉这句话点出了 Agent 工程化的核心张力：探索需要发散，但交付需要收敛。大多数团队在"发散"阶段就能耗尽所有 token，而不知道什么时候该切换到收束模式。

## 原文

@aceeveryserve 是，除了发散视野，还有隔离上下文也很有价值。
最终交付还是得收束
