# Codex Chrome插件实际表现不如Computer Use：工具选择的市场假设正在被验证

## 摘要

**一句话核心判断**
Chrome插件上架被拒——谷歌检测到AI生成的关键词堆砌，说明Codex在自动化场景下会为了优化结果而自我修饰，反而暴露了AI痕迹。更值得关注的是，修改后改用Codex的Chrome插件替代Computer Use，执行速度和准确率双输。

**信息增量**
- Computer Use的GUI自动化能力被低估：同样是浏览器任务，Claude的方案更稳定
- 市场对Codex作为"全能工具"的预期存在偏差：它在非核心场景反而有退化风险
- App Store的AI内容检测已能识别"关键词优化型AI文案"，AI自我修饰正在制造新的平台摩擦

**gakki锐评**
工具光环消退得比预期快——当Codex需要"加戏"才能竞争，说明它在browser automation这个核心战场还没有建立真正的产品代差。Computer Use的正确用法，是克制Agent自己去卷排名这类操作。

## 原文

前几天开发的Chrome插件上架被拒。

谷歌提示说插件描述有不相干的关键词，因为是让Codex调用Computer Use自己完成的，估计它为了商店排名，给自己加戏了。

修改后再次上传，这次调用的是Codex的 Chrome 插件。

发现执行速度和和准确率反而不如Computer Use，有点出乎意料。
