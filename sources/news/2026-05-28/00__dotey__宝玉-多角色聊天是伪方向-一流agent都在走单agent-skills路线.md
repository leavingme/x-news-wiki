# 宝玉：多角色聊天是伪方向，一流Agent都在走单Agent+Skills路线

## 摘要

**[单 Agent 架构已是行业共识，多角色聊天被头部团队放弃]**

宝玉直言多角色聊天（multi-role chat）不是正确方向：大部分场景下单 Agent 足够，所有问题问同一个 Agent，让它自主通过 Skills 和工具获取上下文。

判断依据：如果多角色协作真的靠谱，Codex 和 Claude Code 这些头部 Agent 产品早就做了。

大的方向是单 Agent + 可组合 Skills，复杂任务靠工具调用和上下文组织来实现，而非引入多角色通信的额外复杂度。

gakki：宝玉的逻辑很直接——看头部玩家做什么，而不是看创业公司在说什么。多角色聊天的本质问题是：谁负责协调？谁负责最终决策？每加一个角色就多一层协调成本，而协调成本在大规模任务里是隐形的效率杀手。Skills 模式本质上是在单 Agent 内做能力组合，比多 Agent 通信更可控。

## 原文

需要具体情况具体看，但大的方向是单 Agent，所有问题都问一个 Agent，但是这个 Agent 有不同的 Skills 和一些工具，它可以自主的去决定该怎么得到需要的上下文

多关注 Codex、Claude Code 这些一流 Agent 的设计方向；如果这种多角色聊天靠谱他们早做了
