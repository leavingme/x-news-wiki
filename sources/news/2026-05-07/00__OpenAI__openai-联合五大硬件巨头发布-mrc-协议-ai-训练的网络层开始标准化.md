# OpenAI 联合五大硬件巨头发布 MRC 协议：AI 训练的网络层开始标准化

## 摘要

**OpenAI 联合 AMD、Broadcom、Intel、Microsoft、NVIDIA 发布 Multipath Reliable Connection (MRC) 开放网络协议，目标是减少大规模 AI 训练集群的 GPU 空转时间。**
- 训练效率的瓶颈正在从芯片算力转移到网络通信层
- MRC 是开放协议而非私有标准——这意味着 OpenAI 在基础设施层有了一定的议价权
- 五大硬件巨头同时背书，说明 AI 训练的网络可靠性已是行业级痛点

这是基础设施层面的「修路」行为，短期无感但长期决定训练成本曲线。值得注意的是 OpenAI 在这里扮演了「标准制定者」而非「技术跟随者」的角色——这与其应用层的焦虑形成有趣的反差。

## 原文

We’ve partnered with @AMD, @Broadcom, @Intel, @Microsoft, and @NVIDIA, to release Multipath Reliable Connection (MRC), a new open networking protocol that helps large AI training clusters run faster and more reliably, with less wasted GPU time.
