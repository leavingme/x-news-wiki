# Claude Code的Skill-Creator：A/B测试思维正在重塑AI工具设计基准

## 摘要

**【核心判断】** Claude Code新上线的Skill-Creator功能，将「变量控制」和「对照组思维」产品化，这标志着AI工具设计从「功能堆叠」转向「科学验证」。

- 支持创建「经过Skill处理」vs「原始Prompt」的双路输出对比
- 直接提供修改基准参考，降低Skill调优的认知门槛
- 产品设计层面远超Codex等竞品，将工程思维内化为产品逻辑

gakki 锐评：这不是一个功能更新，这是对AI工具设计哲学的重新定义。当别人还在比谁的模型更强，Anthropic已经在比谁的工具更「聪明地」帮用户用好模型。

## 原文

已经要被claude code在产品层面的能力震惊到说不出来话了，新上的Skill-Creator 可以用来改进用户自己创建的 skill 。其中有一个功能是这样的，直接进行变量控制，根据skill的功能，创建一份prompt，一份经过skill处理，一份不经过skill处理，只用原始的prompt，然后直接对比两份结果，作为下一步修改的基准参考。

我只能说当看到这个功能，我作为 PM 感觉到五体投地，claude code 在产品设计上真的是领先太多身位了，学吧，好好学吧，说的就是你 codex。
