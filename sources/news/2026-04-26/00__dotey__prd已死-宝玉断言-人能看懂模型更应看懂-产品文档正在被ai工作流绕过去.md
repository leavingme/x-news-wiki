# PRD已死：宝玉断言「人能看懂模型更应看懂」，产品文档正在被AI工作流绕过去

## 摘要

**一句话核心判断：** 当 AI 能直接理解产品经理的口语化意图时，PRD 这个中间层正在变成一层需要被人工维护的「技术债务」。

- 核心逻辑：宝玉认为「如果人能看懂，模型更应该能看懂」，并且「几句话直接让 Agent 实现可能更快更好」——这意味着 PRD 的价值不在于「文档本身」，而在于「决策质量」
- 实质是 AI 编程工具成熟度提升后，人机协作层级在压缩：过去「产品→PRD→开发」的三段式，正在被「意图→AI 实施」的两段式侵蚀
- 产品经理的核心价值正在被迫重新锚定：从「写出清晰需求」转向「判断做什么是对的」，后者才真正不可替代

这不仅是工具变化，而是一场关于「谁对结果负责」的组织权责重构。

gakki：「PRD 死不了，死的是写 PRD 的人。产品经理如果只做翻译工作，AI 比你做得更好更快；但如果你做的是判断「做什么」——这个能力现在反而更值钱了。」

## 原文

写 PRD（产品需求文档，Product Requirements Document）要写一版给 Agent 的吗？

我觉得是没必要：
1. 如果人能看懂，模型更应该能看懂
2. 如果产品经理写的 PR 都可以直接给 Agent 用了，大部分场景程序员都不太需要了
3. 其实大部分场景已经不需要 PRD 了，可能几句话直接让 Agent 实现更快更好
