# OpenHuman：个人知识库的正确打开方式，还是Agent外壳的过度包装

## 摘要

**[个人记忆系统终于有了"预设上下文"这个产品思路]**

OpenHuman解决的核心问题：大多数AI Agent需要用户长期使用、反复喂信息后才开始好用。OpenHuman的方案是直接把你的已有文档、Slack、GitHub、Notion、日历等历史资料整理成本地优先的AI记忆系统——Memory Tree切分+SQLite+Obsidian风格vault+三层摘要。

产品成熟度一般，本质是一个"体验不错的个人知识库+Agent外壳"。但产品思路有价值：让Agent跳过冷启动阶段，直接进入"已有上下文"状态——这才是记忆系统应该解决的问题，而不是又一个新的RAG套壳。

同期Web3同名币热度极高，社区在炒作币价而非产品本身。

gakki锐评：个人记忆系统的真正壁垒不在于"能整合多少数据源"，而在于"能不能在保护隐私的前提下让记忆流动起来"。OpenHuman的本地优先方向对了，但若没有明确的隐私边界设计，它只是另一个精致的数据收集器。

## 原文

继 OpenClaw、Hermes 火了之后，又看到有人在捧 OpenHuman。

产品本身解决了一个真实痛点，类似 OpenClaw、Hermes，大多数 AI agent 不认识你，需要你长期使用、反复喂信息后才好用。

OpenHuman 的能更快进入“已有上下文”的状态，它试图把你的已有的文档、Slack、GitHub、Notion、日历等历史资料，整理成一个本地优先的 AI 记忆系统。

它的 Memory Tree 会把内容切成 Markdown 片段，放进 SQLite 和 Obsidian 风格 vault，再生成 source/topic/global 三层摘要。

但事实上，产品成熟度还一般，更准确地说，它是一个体验不错的“个人知识库 + agent 外壳”。

另外，我看到更多的是 Web3 同名币，炒的热度很高，社区也在炒作。。。
