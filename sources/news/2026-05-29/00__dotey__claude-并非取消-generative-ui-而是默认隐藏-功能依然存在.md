# Claude 并非取消 generative UI，而是默认隐藏——功能依然存在

## 摘要

**【Claude generative UI 未死，只是被默认折叠】**

宝玉（@blackanger）明确指出：Claude 的 generative UI 能力并非被移除，而是产品层面选择"默认隐藏"。这一区分至关重要——它意味着该能力已进入稳定可用状态，只是产品入口做了收敛。

- 功能存续证明 Anthropic 对该能力有足够信心，不急于向所有用户开放
- "默认隐藏"是典型的灰度策略：在不破坏用户体验的前提下保留功能
- 社区的"被砍"判断源于产品可见性调整，而非底层能力退化

gakki锐评：产品"隐藏"不等于"删除"，社区情绪往往放大了功能变动的戏剧性。但这个"隐藏"本身值得追问——Anthropic 在为什么样的用户保留这个能力？
