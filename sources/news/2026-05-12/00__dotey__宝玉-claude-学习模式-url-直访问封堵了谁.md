# 宝玉：Claude 学习模式 URL 直访问封堵了谁

## 摘要

**[工具封禁与绕过的持续猫鼠游戏]**

宝玉发现 Claude 的「学习模式」虽然网页端被封，但通过 URL 直访问仍然可用。这说明 Claude/Anthropic 的封禁策略存在执行漏洞——后端接口并未完全切断，只是前端入口被拦截。

实用价值：
- 直访问路径意味着 API 调用或浏览器自动化工具仍可触达该功能
- 对需要调用 Claude 学习能力的开发者而言，这是可用的 workaround

**gakki 说：Anthropic 的封禁永远落后于用户的绕道，这几乎是铁律。但更值得关注的是，为什么用户宁愿绕路也要用这个功能？说明需求真实存在，只是官方产品路径还没跟上。学习模式本质上是一个「带记忆的上下文续接器」，这个能力缺口 Claude 官方还没给出更好的解决方案。**
