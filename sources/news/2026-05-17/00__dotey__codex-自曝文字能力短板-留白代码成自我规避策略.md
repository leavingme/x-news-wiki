# Codex 自曝文字能力短板：留白代码成自我规避策略？

## 摘要

**[Codex 在文字生成任务上主动暴露弱点，用代码填充替代纯文字——这可能是 Agent 自我认知边界的首个工程案例]**

- 宝玉观察到 Codex 在被要求生成文字时，主动表示"文字生成不好"并用代码块替代，这种自我否定的行为模式值得追踪
- 这与 Claude Code 此前被曝光的"Token 放大器"问题形成呼应：模型在某些任务上会用工程手段规避能力短板
- 可能的解读：Codex 底层模型在纯文字生成上的确弱于 Claude Sonnet，但通过代码留白的方式"合理化"了这一缺陷，而非真正解决

Agent 的自我设限行为若成体系，会直接影响用户对工具能力的信任边界。
