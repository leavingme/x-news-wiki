# Codex App 将允许用户远程操控多台 Mac 组成集群，SSH 门槛被彻底抹平

## 摘要

**[多设备远程调用从概念进入 UI 泄漏阶段]**

泄漏的 UI 显示 Codex App 支持跨设备远程操控：用户可在主力 Mac 上远程连接 Mac Mini build 服务器，组成协作集群。关键变化是「不走 SSH，任何人可用 Codex 操控任意多台设备」——这意味着多设备 Agent 编排的门槛从「懂系统配置」降到「会用自然语言」。

- 技术路径：手机/主设备作为控制端，Mac Mini 当 build 服务器，分离构建与执行层
- 潜在影响：个人开发者的算力组合从单机的资源上限中解放
- 边界：远程控制的延迟与稳定性尚未验证，大规模并发场景仍存疑

**gakki锐评：多设备集群化是 AI 编程工具的必然演化方向，但「任何人」能组成集群这句话本身是危险的——当控制权被稀释到对话层级，权限管理与审计盲区也随之放大。Claude Code 若跟进这一路径，安全模型需要重新设计，而不只是加一个远程开关。**

## 原文

第二件事：手机控的不只是自己的 Mac
