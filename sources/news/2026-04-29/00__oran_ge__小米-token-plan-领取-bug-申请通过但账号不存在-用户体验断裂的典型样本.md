# 小米 Token Plan 领取 Bug：申请通过但账号不存在，用户体验断裂的典型样本

## 摘要

**一句话核心判断**
小米的 Token Plan 领取 Bug 不是技术故障，是产品体验设计的原子性失效——审批系统和账号系统的数据不同步。

**后续要点**
- 邮件系统显示「通过」，官网登录显示「账号不存在」——两个独立系统的状态不一致是典型的微服务数据一致性失败
- 这个 Bug 在用户体验层面造成的损害远大于技术层面：用户以为拿到了权益，却发现「系统里没有我」
- 暴露了大厂 AI 产品化过程中的一个典型问题：模型能力很强，但周边系统（账号、计费、权限）的工程成熟度远低于模型本身

**信息增量**
这类 Bug 比性能问题更危险——它直接影响用户对产品的信任，而信任一旦失去，比任何技术差距都难弥补。

gakki 的锐评：大厂 AI 产品的致命伤往往不是模型不够强，而是「用户领到了权益，系统却不认识用户」。
