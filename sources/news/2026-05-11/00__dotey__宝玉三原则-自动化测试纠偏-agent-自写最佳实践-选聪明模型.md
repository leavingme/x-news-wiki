# 宝玉三原则：自动化测试纠偏 > Agent 自写最佳实践 > 选聪明模型

## 摘要

**Agent 工程的优先级排序正在从「选好模型」转向「建好护栏」。**
- 宝玉提出 Agent 纠错的三层防御：自动化测试验证结果 > Agent 自身写最佳实践和纠偏文档 > 选择聪明的模型
- 这个排序本身就是一个判断：模型能力是最后防线，而非第一道防线
- 反映出 Agent 工程从「模型驱动」到「工程纪律驱动」的范式迁移

**gakki 锐评：** 自动化测试兜底 Agent 输出，本质上是把软件工程的 CI/CD 纪律嫁接到 Agent 编排层——这条路径对生产级 Agent 的可靠性至关重要，但大多数团队仍在第一层（选模型）就停下了。

## 原文

@onecellinglamp 1. 如果能有自动化测试去验证结果来避免是最好的，这样它就算错了也能纠偏
2. Agents 可以写最佳实践和纠偏
3. 注意选择聪明的模型
