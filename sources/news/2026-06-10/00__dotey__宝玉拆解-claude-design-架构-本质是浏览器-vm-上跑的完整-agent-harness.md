# 宝玉拆解 Claude Design 架构：本质是浏览器+VM 上跑的完整 Agent Harness

## 摘要

**Claude Design 不只是设计工具，是跑在浏览器+虚拟机上的完整 Agent Harness。**

- 45 个工具 + 24 个内置 Skills，运行机制与 Claude Code 同源（Agentic Loop），不是 Claude Code 的简化版，是另一套独立 Agent 运行时
- 宝玉这轮定位把 Claude Design 从「AI Figma 替代」拽回「垂直 Agent 基础设施」赛道——设计任务只是入口，本质是与 Claude Code 并列的双 Harness 矩阵
- 工具密度极高（45 工具 + 24 Skills）但产品形态聚焦设计场景，说明 Anthropic 在用「场景化 Harness」做用户 Agent 工具链的驯化
- 隐忧：算力与成本仍是开放问题（宝玉自己在另一条回复里承认需要改善），工具堆叠 ≠ 商业可行


gakki 锐评：Anthropic 的阳谋很清晰——用 Design 这个低风险、高频次的入口让用户先习惯 Agent Harness 的运行体感，等用户把「Agent 替你做事」当成日常，再把更重的 Claude Code/Agent 产品塞进来。Claude Design 表面在降维打击 Figma，本质是 Claude Code 的体外预装训练营。

## 原文

Claude Design 有一个完整的 Agent 运行环境，如果把它跟 Claude Code 比较的话，只是它基于浏览器和虚拟机来构建的。它有45个工具，有24个内置skills。运行方式也是基于 Agentic Loop。

所以它就是一个完整的 Agent Harness。 https://t.co/QKnBqSMYSK
