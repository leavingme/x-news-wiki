# GitHub 仓库 VoltAgent/awesome-design-md：设计系统正变成纯文本协议

## 摘要

**【一句话核心判断】**
DESIGN.md 正在从 Google Stitch 的实验变成一个社区共识——设计系统不再需要 Figma 和 JSON，一个 Markdown 文件就是 AI 时代的「设计语言 API」。

**信息增量：**
• 该仓库收集了多个真实产品的设计系统范文，把组件配色、字体体系、布局规范全部拆解成 Markdown 格式
• 作者评价「效果没有特别好，但概念很好」——这是一个诚实的判断，也是一个正在形成的机会
• DESIGN.md 协议的核心价值：AI Agent 读一遍就能仿出 UI，绕过 Figma→JSON→代码的传统翻译链路
• 多工具支持：Claude Code、Cursor、Gemini CLI、GitHub Copilot 均可原生读取

**gakki 锐评：** 这个方向正确但成熟度还不够。真正的突破不是「能不能」，而是「有多像」——当 DESIGN.md 的颗粒度足够细的时候，它会比 Figma 插件更高效，因为它是给 Agent 读的设计系统，而不是给人读的设计系统。

## 原文

VoltAgent/awesome-design-md 这个GitHub仓库收集了很多设计系统范文（DESIGN.md），它把各大网站的设计语言都拆解成markdown格式，包含组件配色、字体体系、布局规范。
虽然目前效果没有特别好，但概念很好 “AI agent读一遍就能仿出UI”。未来应该会做的越来越拟真。
