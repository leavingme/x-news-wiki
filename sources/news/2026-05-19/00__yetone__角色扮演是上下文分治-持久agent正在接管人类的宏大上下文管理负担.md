# 角色扮演是上下文分治：持久Agent正在接管人类的宏大上下文管理负担

## 摘要

**[一句话核心判断]**

Multi Agent 的角色扮演不是过家家，而是应对项目上下文臃肿的工程解法。

**信息增量：**
- 企业 IM（钉钉/飞书）几十年团队协作最佳实践将被 Agent 继承，而非被颠覆
- 角色扮演的深层收益是上下文分治：随项目演进，上下文膨胀不可逆，有角色的持久 Agent 本质是代替人类承担宏大上下文的管理负担
- 轻角色暴露的 Agent Crew 路线被证伪，角色化才是正确方向

**影响：** 对 Multi Agent 系统设计者而言，角色语义隔离 > 任务分解，前者是架构层，后者是执行层。


gakki 说：上下文管理是 Agent 工程的暗物质，yetone 点破了一层窗户纸。但"角色即分治"这个认知本身并不新——它本质上就是微服务的服务边界在 Agent 层的复刻。真正值得追问的是：谁来定义角色边界？human-in-the-loop 还是 self-organized？

## 原文

哈哈，总不能有了钉钉，就不开发飞书了吧。

其实我一开始也很讨厌以企业 IM 的方式思考 Multi Agent / Agent Teams，因为太 boring 太没新意，是一种完全躺平偷懒的做法。后来发现钉钉和飞书这种围绕着企业 IM 的协作形态应该就是这几十年来人类文明在团队协作领域的最佳实践了，未来人类飞升后可能有更妙的实践，但不能以终为始地停在原地等待果实坠落吧！

以前我另一个讨厌 Multi Agent 的点，在于我觉得角色扮演就是在过家家，好羞耻的，所以我一直在 Alma 上实践 Agent Crew 这种轻角色暴露的方式，后来我明白了，角色扮演的一个重大收益就是上下文分治，随着项目的演进，上下文会越来越臃肿和庞大，有角色的且持久化运行的 Agent 们本质上是解放了人类管理宏大上下文的负担。
