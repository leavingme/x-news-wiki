# 宝玉内部消息：TUI和Coding不是护城河，通用Agent的GUI和办公集成才是

## 摘要

**[来自一线开发者的判断：Kimi Code和DeepSeek Harness正站在错误的方向上狂奔]**

宝玉的原话很直接：「卷 TUI 卷 Coding 没前途」。这不是否定Coding能力，而是指出方向问题——当TUI/Coding成为标配而非差异化时，继续堆资源是在做「竞争必答题」而非「战略选择题」。

信息增量：宝玉同时给了出口——「做通用 Agent，尽早支持办公任务」。这是一个有具体落地路径的战略判断，不是空泛的「要做平台」。

这与整个行业狂卷CLI的氛围形成了鲜明反差。

gakki锐评：宝玉这篇文章值得所有在做Coding Agent的团队认真读三遍。「我能做什么」和「市场要什么」之间，隔着一层你可能永远不想承认的墙。

## 原文

Kimi Code、DeepSeek Harness 最好尽早做 GUI，尽早支持好办公任务，做通用 Agent。

卷 TUI 卷 Coding 没前途，当然 Coding 是基础能力，如果 Coding 都做不好其他任务也不会做得好。
