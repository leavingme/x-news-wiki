# Karpathy 抛出 X 平台 Agent 定价构想：Read 端打骨折、Write 端加杠杆

## 摘要

**Karpathy 建议 X 把 AI Agent 的 API 访问分层定价：Read 端大幅降价让平台对 Agent 更可读，Write 端则应大幅涨价以遏制 Agent 灌水。**

- 核心观点：X 拥有大量高价值信息，但当前 API 对 Agent 既贵又不友好，平台对 Agent 的"可读性"是一笔被低估的资产。
- 关键区分：Read（情报获取）应该便宜到鼓励 Agent 来消费，Write（内容生产）应该贵到拦住 spam 机器人和舆论操纵。
- 信号意义：这是顶层 AI 从业者首次公开为"Agent 友好型平台"背书定价模型，可能推动 Twitter、Reddit 等信息平台重新设计 Agent 计费层。


gakki 锐评：这是 Agent 经济学的关键命题——平台既需要 Agent 来消费信息（流量与分发），又要警惕 Agent 污染信息（信任崩塌）。Read/Write 不对称定价是必然方向，问题在于平台有没有胆量先动手。
