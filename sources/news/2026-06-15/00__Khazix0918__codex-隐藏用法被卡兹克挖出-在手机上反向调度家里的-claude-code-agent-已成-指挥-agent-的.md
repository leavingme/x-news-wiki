# Codex 隐藏用法被卡兹克挖出：在手机上反向调度家里的 Claude Code，Agent 已成「指挥 Agent」的指挥塔

## 摘要

**[卡兹克实测揭示 Agent 工具链的真实使用形态：Codex 已从「编码工具」退化为「Agent 指挥入口」，Claude 官方客户端体验反成短板]**

- 数字生命卡兹克公开用法：手机端用 Codex 远程启动家里电脑上的 Claude Code，并开启远程控制继续 coding。
- 顺带吐槽：Claude 官方 Dispatch 客户端体验极差，整个客户端做得很垃圾。
- 信号：用户实际把 Codex 当作 universal orchestrator 入口来用——一个 Agent 调度另一个 Agent 已是日常。
- 讽刺点：Claude 自己的客户端被嫌弃，分发被竞品 Codex 接管。


gakki 锐评：Codex-as-orchestrator 的真实落地场景比 OpenAI 官方叙事更野——用户不是拿它写代码，是拿它当 Agent 指挥塔跨设备跨工具调度 Claude。Anthropic 在工具链分发上被 OpenAI 反向吃掉一层，这跟 Ona 收购的「超级入口」战略形成共振：官方客户端没跟上用户已经长出来的使用形态。
