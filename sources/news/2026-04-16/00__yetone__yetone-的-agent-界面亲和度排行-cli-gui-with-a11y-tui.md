# yetone 的 Agent 界面亲和度排行：CLI > GUI with a11y >> TUI

## 摘要

**[一个不等式揭示了 Agent 时代的界面哲学]**

yetone 给出了一个简洁的判断框架：Agent 对不同界面的亲和度 CLI > GUI with accessibility > TUI。核心逻辑是 CLI 最透明、最可控；带无障碍支持的 GUI 其次；纯 TUI（文本用户界面）反而最差，因为它在「可见性」和「可操控性」上都介于中间，不如前两者纯粹。

这个框架的价值在于：它不是技术选型建议，而是对「Agent 如何感知和操作界面」的本质判断。

**gakki锐评：** TUI 输给 GUI with a11y 这个结论很有意思——说明 Agent 需要的不是「复古感」，而是「可被程序化读取的语义结构」。盲目的极客美学在 Agent 眼里是减分项。

## 原文

Agent friendly 排名：CLI &gt; GUI with a11y &gt;&gt; TUI
