# "你别问对不对，你就说快不快"——JS 排版方案的暴力美学反思

## 摘要

**[ 用近似解换取速度的工程哲学，正在被 AI Agent 重新包装 ]**

评论指出某方案用 JS 代码逼近浏览器未暴露的文字排版 API，忽略复杂排版行为，牺牲正确性和泛用性——本质上是「快就行」的暴力美学。

- 牺牲正确性换速度的思路，在 AI Agent 时代被包装成新范式
- 这个问题在浏览器 Agent 场景下有普遍性：结构化操作 vs 渲染保真度的永恒张力

当 AI 开始「差不多得了」的时候，工程纪律的边界正在被重新划定。这不只是技术选择，是 Agent 工程的价值观问题。


## 原文

用 JS 代码逼近浏览器没有暴露的文字排版 API，忽略大部分复杂的排版行为，牺牲正确性和泛用性。这不就是又一个 “你别问对不对，你就说快不快吧” 的东西么？
