# 向阳乔木：飞书CLI是目前最全面的Agent工具链，超越小龙虾/Hermes/Codex/Claude Code

## 摘要

**一句话核心判断**
飞书CLI被推荐为覆盖最全面的CLI工具，但这个判断本质是个人偏好而非行业共识，对象感错位。

**信息增量**
- 飞书CLI安装量和支持工具链的完整性获得开发者认可
- 但「最全面」的定义取决于使用场景，飞书生态外的适用性存疑

**影响评估**
- 飞书在Agent工具链的整合上找到了一条差异化的入口
- 对国内开发者有参考价值，海外开发者生态可替代方案更多


gakki锐评：「最全面」是危险的说法，取决于你在谁的生态里。飞书CLI的优势是飞书生态，跨出飞书边界，它和小龙虾、Hermes的竞争维度完全不同。这个推荐对飞书用户有价值，但不应泛化为行业判断。

## 原文

如果只允许给小龙虾、Hermes、Codex、Claude Code 装一个 CLI，那一定选飞书 CLI，实在是太全面了。

安装指令：npx @larksuite/cli@latest install
