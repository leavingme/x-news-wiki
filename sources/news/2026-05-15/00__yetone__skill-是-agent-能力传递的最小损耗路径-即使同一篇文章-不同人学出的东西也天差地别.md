# Skill 是 Agent 能力传递的最小损耗路径——即使同一篇文章，不同人学出的东西也天差地别

## 摘要

**能力传递的"最后一公里"问题。**

同样一篇文章链接，不同人给 Agent 的上下文截然不同：有人只丢链接让 Agent 自己消化，有人会补充元提示让 Agent 逆向真实 App 获取更深层知识。再加上 harness、模型型号、reasoning effort、提示词模板的差异，同一份"能力"在不同人手里的输出质量可能相差一个数量级。

Skill 解决了这个问题：把能力封装为可复制的制品，让"学完这篇文章并亲自逆向真实 App"这个完整路径变成一条可迁移的指令，而不只是口头描述。

gakki：Skill 的本质不是"分享提示词"，而是"固化行为路径"。这也是为什么从 SKILL.md 到 Anthropic 的 Skills 生态，核心壁垒从来不在格式而在 know-how 的凝结密度。

## 原文

因为每个人用 agent 的能力和习惯是不一样的，就像你上面的推文提到的，你只会直接把文章链接发给 Agent 让它去学习，而我不仅会把这篇文章链接发给 Agent 让它去学习，而且还会强调一下让 Agent 学习完这篇文章后亲自逆向一下安装在本机的 Raycast Beta app 来根据真实世界的代码进一步提取更深的洞见和知识，最后让 Agent 把这一切凝结成了这个 skill。你会觉得你我这两个不同的行为得到结果会是一样的吗？更何况不同的人用的 harness 、模型型号、reasoning effort、提示词等等都千差万别，那如何让一个能力以最小的损耗和最准确的状态触达更多的人群呢？skill 无疑是最好的选择。
