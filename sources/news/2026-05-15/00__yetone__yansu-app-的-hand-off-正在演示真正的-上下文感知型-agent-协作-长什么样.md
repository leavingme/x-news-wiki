# Yansu App 的 Hand-off 正在演示真正的「上下文感知型 Agent 协作」长什么样

## 摘要

**[Yansu 的 Hand-off 功能已展示出上下文状态继承与主动建议的 Agent 原生交互范式]**

1. Agent 主动监测用户行程，在检测到旅行场景后主动发出 Airbnb 调研邀请——这是"预测式 Agent 协作"而非响应式交互
2. Hand-off 时自动继承近期会议语言（中文/英文），跨语言场景无需用户显式指定上下文
3. 更关键的是：用户分享推文意图出现后，Agent 立即主动询问"是否需要帮你写这条推文"并直接执行——意图识别→行动路径压缩已经压缩到单轮交互

Yansu App 正在演示一个具体可复制的 Agent 原生协作样本，比市面上的"多 Agent 框架"讨论更有产品落地价值。

**gakki锐评：** Hand-off 功能本质上解决的是"Agent 之间谁来发起、谁来承接"的决策权问题——Yansu 选择了由 Agent 主导发起、用户确认后执行的模式。这个设计方向对不对不知道，但它比当前大部分多 Agent 系统里"人工写 prompt 协调"的方案更接近真正的自主协作。值得持续跟踪。

## 原文

说起我们的 Yansu App 的真实案例，不得不提 Yansu 的 Hand-off 功能，最近发生了两个特别好玩的事情：

首先，我们的 CEO Bo @bozhao  最近在准备旅行，Yansu 看到这个信息之后，主动向 Bo 发出了一个 Hand-off 的消息提醒，说想要帮他去调研目的地的 Airbnb 房源。Bo 点击确认后，它果然给出了一个质量特别高的房源推荐列表。

Bo 是个英文用户，但最后 Yansu 是用中文总结的。Bo 问它：「你为什么用中文总结？」最后发现是因为在它做 Hand-off 之前，Bo 刚跟我们开了一个全中文的会议，所以它就延用了中文进行总结。

这件事情还没完。在这之后 Bo 刷推看到了我现在引用的这条推文，他就想把上面这个案例给分享出来，有意思的是，Yansu 紧接着就发起了一个新的 Hand-off，问道：「是否需要我把 Yansu 的这个案例推文帮你写出来？」，Bo 点击 ok 后，Yansu 一气呵成写出了推文。

这两件事情真的震惊了我们。
