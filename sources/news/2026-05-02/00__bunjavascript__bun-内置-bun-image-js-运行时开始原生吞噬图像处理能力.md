# Bun 内置 Bun.Image：JS 运行时开始原生吞噬图像处理能力

## 摘要

**Bun 下个版本将内置多格式图像处理库，开发者的图片操作不再依赖 Sharp/ImageMagick 等外部依赖。**

- Bun.Image 是 builtin 原生模块，性能预期远超 Node.js 生态的现有方案
- 对 AI 工作流的意义：Agent 需要频繁处理截图、图表、多模态素材，原生图像能力减少依赖链
- JS/TS 运行时正在从「Web 后端」向「全栈 AI 工程基建」进化

gakki 锐评：Bun 的策略很清晰——把 AI 工程师常用的能力全部内置。当你的 Agent 需要裁图、缩图、转格式时，少装一个 npm 包就是少一个故障点。
