# 宝玉的工程实话：项目进度紧时，传统开发仍比 Vibe Coding 保险

## 摘要

**AI Coding 不是银弹——交付压力下，传统可控流程依然是工程默认值。**

- 宝玉直言"项目进度紧的时候还是传统方式最保险"，是 Vibe Coding 热潮里的反向声音
- 隐含判断：AI Coding 的"快"是平均值优势，但在 deadline / 风险敏感场景下，确定性 > 速度
- 与当前"零人工代码交付"叙事形成对照，提醒工程纪律不能被速度幻觉吃掉
