# 微信好友触1万上限后,wx-cli 类工具正在把'删除联系人'变成 Agent 自动化任务

## 摘要

**微信私域管理正在被 CLI 工具接管:'加好友时间/是否聊过/共同群'三个维度被封装成 Agent 决策依据,原本依赖人感的清理动作变成可批量执行的任务。**

- 信号点:向阳乔木一句'用什么原则删除'反映出需求从'个人感觉'走向'规则化筛选 + 自动化执行'
- 价值点:wx-cli 把'删除联系人'这种高摩擦、高情绪成本的私域动作变成 CLI 流水线,类似 Notion/飞书里的批量整理脚本
- 边界:仍是个人脚本阶段,未见通用产品形态;读取聊天记录、共同群关系涉及隐私边界,规模化前需要合规框架


gakki 锐评:'好友过载'是 2026 中产数字生活的标准症状,Agent 接管是必然——但谁会做出'微信好友清理助手'这个 2C 产品,以及腾讯会不会在客户端层面自己吃掉这个需求,才是值得跟踪的开放问题。

## 原文

微信满一万好友上限后，都用什么原则删除不相干的人啊？

之前朋友给我演示他用wx-cli写的工具，看什么时间加的，有没有聊过，有没有共同群做判断。

各种小秘书、助手、客服都删了，还是不够用。
