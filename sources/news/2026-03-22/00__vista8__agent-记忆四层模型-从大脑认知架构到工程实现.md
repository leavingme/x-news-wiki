# Agent 记忆四层模型：从大脑认知架构到工程实现

## 摘要

**[把神经科学的睡眠机制搬进 Agent 架构]**

• 工作记忆 ↔ 上下文窗口：保持清爽，防止过载
• 程序性记忆 ↔ Skill/Tools：自动化调用，无需显式思考
• 情景记忆 ↔ JSONL 日志：会话结束后由「海马体」索引整理
• 语义记忆 ↔ MEMORY.md：长期知识沉淀，跨会话持久化

gakki 锐评：这个类比的价值在于把「记忆系统」从黑盒配置变成可设计的认知架构——Agent 也需要「睡觉」来巩固记忆。

## 原文

Claude Code中的Agent记忆系统，AI给的类比，感觉像这么回事儿：

① 上下文窗口 = 工作记忆
② Skill/Tools = 程序性记忆
③ JSONL对话记录 = 情景记忆
④ Memory/Claude.md = 语义记忆

好的Agent 记忆架构像睡眠充足的大脑：

① 工作记忆保持清爽（上下文不爆）
② 自动化的交给小脑（技能按需调用）
③ 睡觉时海马体做整理（会话结束后索引日志）
④ 醒来时皮层知识已更新（MEMORY.md 注入新事实）
