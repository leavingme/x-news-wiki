# Alma 用自写 ReAct XML Parser 补 DeepSeek V4 Tool Calling 短板：第三方中间件的机会窗口

## 摘要

**[一句话核心判断]**

DeepSeek V4 自身 API 的 Tool Calling 不稳定，被 Alma 通过外部手撸 ReAct XML Parser 方式绕过去——这揭示了一个正在形成的中间件市场。

**信息增量/影响：**
- 模型能力≠产品能力：底层模型缺陷正在被第三方适配层填补，这是工程栈迁移的机会
- ReAct XML Parser 成为可复用组件，暗示 Tool Calling 标准化程度仍极低
- 小团队/独立开发者可通过"缝合"而非"训练"建立差异化壁垒

**gakki 锐评：** DeepSeek 的 API 稳定性问题若持续，会催生一个类似"模型网关"的中间件生态——这比做模型本身更稳赚。

## 原文

在 Alma 中可以为 DeepSeek V4 模型手动开启 Tool Calling，这样就会用手撸的 ReAct XML parser 来解决 DeepSeek Provider 本身 API 的 Tool Calling 不稳定的问题
