# OpenClaw 的「墙外效应」：Web Search 成为绕不过的基础设施

## 摘要

**[境外 LLM 与本地部署的地缘张力]**
• 全民 OpenClaw 意味着全民获得境外 LLM + Web Search MCP 的「组合拳」，本地 VPN 搭建门槛被击穿
• 国产 LLM 供应商面临两难：断 Web Search 则 Coding 场景失效，不断则用户可借道获取境外 endpoint
• 百度独占搜索结果的「残废版」套餐可能成为最终妥协，用户被迫在可用性与合规性之间做选择

**gakki 锐评：** 技术封锁在 Agent 工具链面前像个筛子——只要允许写代码和搜索，就没有封得住的墙。国产大模型的「温室」正在漏风。

## 原文

全民openclaw的结果，

就是全民拿到境外LLM、境外web search MCP，

靠这两个东西，就完全足够用openclaw在本地搭建一个VPN并且获得最及时、尚未封掉的endpoint，

我甚至觉得，这东西在国内几家LLM供应商都封不住，因为web search是不能断的，否则普通coding用户、claude code用户就用不了了，

现在最严峻的问题是，既然要给用户提供web search，又不能让用户drive openclaw去配置本地VPN然后用web search找到endpoint，

估计最终结果就是，国产套餐web search仅限百度，搜索结果大幅度下降，百度收录和搜索英文等于半个残废，用户只能捏鼻子忍一忍。
